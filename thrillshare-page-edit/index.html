What This Tool Does

Finds the Student Resources and Parent Resources pages on every EUSD school's Thrillshare CMS site — even though those pages are named inconsistently across schools (e.g. "Students", "Resources for Students", "Parents and Families", "Information for Families") — and adds an accessible text caption beneath every graphical button on those pages. The caption is a centered text link that points to the same destination as the graphic, so screen-reader and keyboard users get a real, selectable label.

Why this matters: Image-only navigation buttons fail WCAG 2.1 and were flagged in the EUSD DOJ Web Rule audit. Adding a captioned text link preserves the visual design while giving assistive tech something meaningful to announce.

The tool is idempotent — re-running it on a page that already has captions will detect that and skip it without making any changes.

Prerequisites

One-Time Setup

  1. Python 3.10+ installed
  2. Playwright with Chromium: playwright install chromium
  3. Project folder: ~/projects/eusd/thrillshare-page-edit/
  4. Thrillshare access — an EUSD Google account (cc2@eusd.org) with CMS edit permissions
  5. Gemini API key — required for OCR on button graphics (see Configuration below)

If Setting Up on a New Machine

CLI Reference

python edit.py [MODE] [OPTIONS]

Modes (mutually exclusive)

FlagWhat It Does
--login Opens a headed (visible) Chrome browser, prompts you to log in via Google SSO, saves cookies to session/cookies.json, then exits. Run this once before any other command, or whenever your session expires.
--discover-only Lists all Resources pages found across all schools and exits — no edits are made. Useful for verifying discovery before a full apply.
--dry-run (default) Applies captions to one chosen page, then opens the page in a browser for visual verification. Requires --school. Safe to use at any time.
--apply Applies captions to all discovered pages across all schools. Writes a report to reports/. This modifies live pages.

Options

OptionUsed WithPurpose
--school NAME--dry-runExact school name to target (required for dry-run). Run --discover-only first to see valid names.

--no-browser-verify--dry-runSkip opening the page in a browser after the dry-run edit. Useful in headless/scripted contexts.

Typical Workflow

1 Log in (first time only, or after session expiry)

python edit.py --login

A Chrome window opens. Sign in with your EUSD Google account. Once the CMS loads, cookies are saved and the window closes automatically.

2 Discover pages (optional sanity check)

python edit.py --discover-only

Prints a table of every Student/Parent Resources page found, with school name, page title, page ID, and edit URL. Expect 20+ entries (two per school that has both pages).

3 Dry-run one school to verify the output

python edit.py --dry-run --school "Conway Elementary"

Edits just that school's Resources page(s) and opens the page in your browser. Eyeball the captions, then press Ctrl+C to finish.

# If a school has both Student and Parent pages, target one specifically:
python edit.py --dry-run --school "Conway Elementary" --page "Student"

4 Apply to all schools

python edit.py --apply

Runs a two-pass process across all schools, writes a report to reports/, and prints a summary. See Reading the Report below.

How Captions Are Generated

For each image button, the tool determines the caption text using this priority:

  1. Gemini OCR — downloads the button graphic and asks Gemini Flash to read the visible text. This is the primary path for most EUSD buttons, which have text baked into the PNG.
  2. URL-slug fallback — if OCR fails or returns no text, derives a label from the last segment of the link URL (e.g. /page/calendars-and-schedules → "Calendars and Schedules").

The report marks URL-slug captions with * — review these manually as the URL slug may not match the visible button text.

Cross-School Link Inference

Some schools have image buttons with no link URL configured (the image exists but doesn't go anywhere). During --apply, the tool runs a two-pass process:

  1. Pass 1: Fetches all pages and builds a map of caption text → destination URL from schools where the same button is properly linked.
  2. Pass 2: For each unlinked button, OCRs the graphic and looks up its caption text in the map. If a confident unique match is found, the inferred URL is used. If no match or multiple conflicting matches exist, the button is flagged as unresolved in the report for manual review.
Unresolved buttons are listed at the bottom of the report. These are buttons with no link URL that the inference couldn't resolve. They must be fixed manually in the CMS.

Reading the Report

After --apply, a timestamped text report is saved to reports/ and printed to the terminal. The report has four sections:

SectionMeaning
EDITED (n)Pages where captions were successfully added. Each page lists its captions with source (gemini or url-slug). Captions marked * used URL-slug fallback — review these. Lines marked ! had their link URL inferred from another school — verify the URL is correct.
SKIPPED — already complete (n)Pages where all buttons already had captions. No changes were made.
ERRORS (n)Pages where something failed (fetch error, write error). These were not edited.
UNRESOLVED (n)Image buttons with no link URL that couldn't be inferred from other schools. Requires manual CMS fix.

Troubleshooting

ProblemFix
Could not authenticate. Run with --login.Your session cookies have expired. Run python edit.py --login to re-authenticate.
No matching page for school=...Use --discover-only to see the exact school name as returned by the CMS (spelling and capitalization must match).
ModuleNotFoundError: No module named 'playwright'Activate your virtual environment first: venv\Scripts\activate, then try again.
GEMINI_API_KEY not setAdd your Gemini API key to .env. The tool won't be able to OCR button graphics without it.
Report shows many url-slug captionsThe Gemini OCR couldn't read the button text (possibly a non-PNG format or very stylized image). Review those captions manually and edit in the CMS if needed.
Unresolved buttons in reportThese buttons have no link URL and no other school has a matching captioned button to infer from. Open the page in the CMS and add the correct link manually.
