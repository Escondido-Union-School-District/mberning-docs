PDF Form Prep Toolkit

User Guide — Escondido Union School District

Last updated: 2026-03-25 — Initial guide created

What This Toolkit Does

This toolkit automates accessibility remediation of Word-exported PDF forms. When you create a form in Word using underscores for fill-in lines (e.g., "First name:________________"), the label and underscores end up fused together in the PDF tag tree, making screen readers read them as one block. Fixing this manually in Acrobat's Reading Order tool is tedious.

The toolkit has two scripts that handle different stages of the process:

ScriptWhat It DoesWhen to Run
prep_form.py Splits label text from underscore fill lines in the PDF tag tree. Underscores are moved to Artifact blocks so screen readers skip them. Cleans up empty tags. After exporting the form from Word to PDF
post_prepare.py Tags form field annotations, generates WCAG-compliant tooltips using Gemini AI, and creates proper <Form> structure elements. After running Acrobat's "Prepare Form" on the prepped PDF
Requirement: Input PDFs must be tagged PDFs exported from Word. Untagged or scanned PDFs will not work.

The Full Workflow

StepToolAction
1WordCreate form with underscore blank lines
2WordExport to PDF (Save As → PDF)
3prep_form.pySplit labels from underscores, artifact the underscores
4AcrobatOpen prepped PDF → Prepare Form → fix field types only
5post_prepare.pyTag fields, generate tooltips, create <Form> tags
6AcrobatVerify result

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.10+ must be installed
  2. Project folder: C:\Users\mberning\projects\eusd\pdf-form-prep
  3. Dependencies installed via pip
  4. Google API key saved in the .env file (required only for post_prepare.py)

If Setting Up on a New Machine

# 1. Clone or copy the project folder

# 2. Open a terminal in the project folder and install dependencies:
pip install -r requirements.txt

# 3. Create the .env file (only needed for post_prepare.py):
#    Copy .env.example to .env, then add your Google API key:
GOOGLE_API_KEY=your-actual-api-key-here
Getting a Google API Key: Go to aistudio.google.com, sign in, click "Get API key", and create a key. The free tier allows 15 requests per minute, which is sufficient for tooltip generation.

Step-by-Step: prep_form.py

This script handles Step 3 of the workflow: splitting labels from underscores.

1 Place the Word-exported PDF in the input\ folder

C:\Users\mberning\projects\eusd\pdf-form-prep\input\

2 Open a terminal and navigate to the project

cd C:\Users\mberning\projects\eusd\pdf-form-prep

3 Run the script

python prep_form.py "input/Your Form.pdf"

4 Check the output

The prepped PDF is saved to output\ with _prepped appended:

output/Your Form_prepped.pdf

You'll see a summary like this:

[Step 1] Splitting label text from underscores...
  Page 1: split MCID 3
  Page 1: split MCID 4
  Page 1: split MCID 6 (3 segments)
  ...
[Step 2] Cleaning up empty tags...
  Removed 1 empty tags

=== Summary ===
Labels split:      21
Empty tags removed: 1
Output:            output\Your Form_prepped.pdf

5 Open the prepped PDF in Acrobat

Run Prepare Form (Tools → Prepare Form). Acrobat will auto-detect form fields. Fix any wrong field type guesses:

  • Change text fields that should be checkboxes
  • Change text fields that should be radio buttons
  • Do NOT fix titles, tooltips, or structural tags — post_prepare.py handles those

Save the PDF (e.g., to input/Your Form_prepared.pdf).

Step-by-Step: post_prepare.py

This script handles Step 5: tagging fields, generating tooltips, and creating structure.

1 Run the script on the Acrobat output

python post_prepare.py "input/Your Form_prepared.pdf"

2 Review the proposed tooltips

The script matches each form field to its label, generates tooltips via Gemini AI, and shows you the proposals:

=== Proposed Titles and Tooltips ===

  [1] Text "First name"
      TU: "Enter your first name"
  [2] Text "School"
      TU: "Enter the name of the student's school"
  [3] Checkbox "No aligned materials"
      TU: "Select this option if a pupil does not have..."

Accept all? (y/n/edit #):
  • y — accept all proposals and proceed
  • n — abort without saving
  • edit 3 — re-prompt Gemini for field #3 with additional guidance

3 Check the output

The final PDF is saved to output\ with _final appended:

output/Your Form_prepared_final.pdf

The --skip-tooltips Option

If you only want structural tagging (no Gemini tooltip generation), use:

python post_prepare.py "input/Your Form_prepared.pdf" --skip-tooltips

This skips the API calls and interactive review. Useful if you want to add tooltips manually in Acrobat.

Understanding the Output

prep_form.py reports how many labels were split from underscores and how many empty tags were cleaned up. Each split means one form field label was successfully separated.

"3 segments" means a multi-field line like "City:___ State:___ Zip:___" was split into 3 separate labels.

post_prepare.py reports fields processed by type, how many were matched to labels, tooltips generated, and WCAG compliance check results.

Troubleshooting

ErrorCauseFix
Error: PDF is not tagged The PDF wasn't exported as a tagged PDF from Word Re-export from Word: File → Save As → PDF → Options → check "Document structure tags for accessibility"
Error: file not found Wrong path or filename Check the filename and use quotes around paths with spaces
PermissionError The output file is open in Acrobat Close the PDF in Acrobat and run again
GOOGLE_API_KEY not set Missing or empty .env file Create/edit .env with your API key (only needed for post_prepare.py)
ResourceExhausted / rate limit Too many Gemini API calls too fast Wait a minute and try again. The tool has built-in 4-second delays between calls.
No fields split (0 splits) PDF doesn't use underscore fill lines, or underscores are drawn as lines rather than text characters Check that the form uses actual underscore characters (_) in Word, not tab leaders or drawn lines
No form fields found Acrobat's Prepare Form wasn't run before post_prepare.py Open the prepped PDF in Acrobat, run Prepare Form, save, then run post_prepare.py
Fields not matched to labels Field names don't resemble any label text These are logged as warnings. Check manually in Acrobat.
