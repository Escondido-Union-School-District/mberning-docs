Operational reference for Mark Berning's EUSD project automation

Daily Schedule

TimeWhatWhereWhat it does
6:00 AM Morning Digest Email Your PC (Windows Task Scheduler) Sends cross-project summary email of yesterday's activity to mberning@eusd.org. Logon rescue trigger fires if PC was off.
6 AM–4 PM (hourly) Thrillshare Approval Checker Your PC (Windows Task Scheduler) Checks all 28 EUSD schools for pending Thrillshare approvals. Always sends email — subject line indicates whether approvals are pending.
7:00 AM Thrillshare Smore Link Scanner Your PC (Windows Task Scheduler) Checks Thrillshare alerts for principals using Smore links instead of Thrillshare alerts. Always sends email. 36-hour lookback.
2:50 PM Afternoon Automation Your PC (Windows Task Scheduler) Captures session notes with reasoning per project, updates user guides, syncs to GitHub Pages, generates daily summary.
2:55 PM Config Sync Your PC (Windows Task Scheduler) Backs up Claude Code config (skills, settings, profiles) to git.

How to Start a New Project

  1. cd ~/projects/eusd and start claude
  2. Explore the idea conversationally — take as long as you need
  3. When ready to build, tell Claude (any phrasing: "let's build it", "set this up", "create the project")
  4. Claude creates the directory, git, GitHub repo, aliases, user guide, and captures your reasoning
  5. Open a new terminal and type the alias Claude gave you
Note: If you skip these steps and just create a directory and start coding, the system still works. The afternoon automation discovers new projects automatically. You just get less reasoning captured.

New Project Gut Check

CheckHow to verifyWhat to tell Claude if missing
Git initialized? .git/ exists in project root "Initialize git for this project"
CLAUDE.md exists? CLAUDE.md in project root "Create a CLAUDE.md for this project"
GitHub repo created? git remote -v shows a remote "Create a GitHub repo for this project"
Launch alias set up? Type project initials in a new terminal "Add a launch alias for this project"
Reasoning captured? CLAUDE.md has a Background section, or docs/superpowers/specs/ has a design spec "Write up why we're building this, what approach we chose, and what alternatives we considered — put it in CLAUDE.md"
User guide started? docs/user-guide.html exists "Create a user guide for this project"
If you can only do one thing: capture the reasoning. Everything else is automatic or can be backfilled later.

How to Come Back to a Project

  1. Type the project alias (see inventory below)
  2. Say "where were we"
  3. Claude reads session notes, design rationale, and git history, then briefs you on what you were doing, why, and what's next
