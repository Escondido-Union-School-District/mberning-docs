What This Tool Does

This tool has two modes that automate Thrillshare monitoring:

ModeFlagWhat It DoesSchedule
Approval Checker(default)Checks all schools for pending approvals in Live Feed, News, and CMS pagesHourly on weekdays
Smore Link Scanner--smore-scanScans specific principals' emails for Smore newsletter linksDaily at 7:00 AM

Both modes use browser automation (Playwright), log into Thrillshare via saved session cookies, and email results only when something is found.

Note: This tool uses browser automation (Playwright) to interact with Thrillshare. It requires an initial manual Google SSO login to establish a session cookie, which is then reused for automated headless runs.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.9+ installed
  2. Project folder: C:\Users\mberning\projects\eusd\thrillshare-checker
  3. Virtual environment with playwright and python-dotenv installed
  4. Chromium browser installed via Playwright (playwright install chromium)
  5. Environment file (.env) configured with SMTP and email settings
  6. Session cookies saved from an initial --login run

If Setting Up on a New Machine

# 1. Clone or copy the project folder

# 2. Create and activate a virtual environment
python -m venv venv
venv\Scripts\activate

# 3. Install dependencies
pip install -r requirements.txt

# 4. Install the Chromium browser for Playwright
python -m playwright install chromium

# 5. Create the environment file
copy .env.example .env
# Edit .env with your SMTP and email settings

# 6. Run the first-time login to establish a session
python checker.py --login
Warning: The .env file contains email configuration. Never commit it to version control. A .env.example template is provided as a reference.

Step-by-Step

The tool has two modes: an initial interactive login and the regular automated check.

1 First-Time Login

Run with the --login flag to open a visible browser window. Log in through Google SSO when prompted. The script saves your session cookies so future runs work without manual login.

cd C:\Users\mberning\projects\eusd\thrillshare-checker
venv\Scripts\activate
python checker.py --login

A Chromium window will open. You'll see a prompt:

=== LOGIN REQUIRED ===
Please log in using Google in the browser window.
Take your time — the script will wait.

Complete the Google SSO login. Once done, the script saves your cookies and proceeds to check all schools.

2 Run an Automated Check

After the initial login, run without flags for a headless check. If pending items are found, an email is sent automatically.

python checker.py

3 Run Without Sending Email

To see results in the terminal without sending email (useful for testing):

python checker.py --no-email

4 Scheduling (Optional)

The tool can run automatically via Windows Task Scheduler. VBS wrappers run the checker silently (no console window) and write output to log files.

Two scheduled tasks are configured:

Task NameScheduleWrapperLog File
ThrillshareCheckerHourly, weekdays 6 AM – 4 PMrun_hidden.vbslog.txt
Thrillshare Smore ScanDaily at 7:00 AMrun_smore_scan.vbssmore_scan_log.txt

Step-by-Step: Smore Link Scanner

The Smore scanner finds emails sent by specific principals that contain Smore newsletter links. It navigates to each school's Media > Alerts > Analytics page, filters by the principal using the "Published By" dropdown, and extracts any app.smore.com URLs from recent messages.

1 Run a Manual Smore Scan

cd C:\Users\mberning\projects\eusd\thrillshare-checker
venv\Scripts\activate
python checker.py --smore-scan --no-email

2 Run With Email Notification

Drop the --no-email flag. An email is sent only if Smore links are found.

python checker.py --smore-scan

3 Review the Report

CLI Reference

FlagDefaultDescription
--loginOffForce a visible browser login. Use for first-time setup or when your session expires.
--no-emailOffPrint results to the terminal only. No email is sent regardless of findings.
--smore-scanOffScan configured principals' emails for Smore links instead of checking approvals. Covers the last 36 hours.

With no flags, the tool runs headless and sends email only if pending approvals or errors are found.

Troubleshooting

ErrorCauseFix
Session expired. Run with --login to re-authenticate. Saved cookies have expired or become invalid Run python checker.py --login and complete the Google SSO login
No schools discovered. The organization dropdown wasn't found on the page, or the page didn't load correctly Run with --login to see the browser. Verify you can see the school switcher dropdown. If the Thrillshare UI changed, selectors in checker.py may need updating.
Could not find MEDIA nav link! The MEDIA navigation link wasn't found after logging in Run with --login to check whether you land on the accounts page. Ensure your Google account has Thrillshare media access.
Failed to send email: ... SMTP relay connection failed Check your .env settings. Verify SMTP_HOST (default: mail.eusd.org) and SMTP_PORT (default: 25) are correct. Ensure the machine can reach the mail server on the network.
Timed out waiting for login. The 2-minute login timeout was reached before SSO completed Run with --login again and complete the Google login more quickly. The script waits up to 2 minutes.
Scheduled task runs but no email and no report The VBS wrapper or paths may be misconfigured Check log.txt for error output. Verify the paths in run_hidden.vbs match your actual project location.
Error checking CMS: ... The CMS pages section failed to load or the table structure changed Run manually with --no-email to see the full error. The CMS check navigates directly to thrillshare.com/cms/v2/pages — verify that URL works in a normal browser.

Key Files

  • .env — SMTP host, port, sender/recipient email
  • session\cookies.json — saved login session
  • smore_scanner.py — principal list (edit SMORE_SCAN_SCHOOLS to add schools)
  • reports\ — date-organized report history (auto-cleaned after 3 days)
  • log.txt / smore_scan_log.txt — output from scheduled runs
