What This Tool Does

This tool has two modes that automate Thrillshare monitoring:

ModeFlagWhat It DoesSchedule
Approval Checker(default)Checks all schools for pending approvals in Live Feed, News, and CMS pagesHourly on weekdays
Smore Link Scanner--smore-scanScans all schools for Smore links in messages from the last 7 days (168h); also re-checks discovered links at ~24h and ~48h and emails a diff if the page changedWeekdays (Mon–Fri) at 3:00 AM and 6:50 AM

Both modes use browser automation (Playwright), log into Thrillshare via saved session cookies, and always send an email after every run — subject line reflects the result ("Pending Approval" vs "Check Complete — No Pending Approvals").

Note: This tool uses browser automation (Playwright) to interact with Thrillshare. It requires an initial manual Google SSO login to establish a session cookie, which is then reused for automated headless runs.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.9+ installed
  2. Project folder: C:\Users\mberning\projects\eusd\thrillshare-checker
  3. Virtual environment with playwright and python-dotenv installed
  4. Chromium browser installed via Playwright (playwright install chromium)
  5. Environment file (.env) configured with SMTP and email settings
  6. Session cookies saved from an initial --login run

Step-by-Step

The tool has two modes: an initial interactive login and the regular automated check.

1 First-Time Login

Run with the --login flag to open a visible browser window. Log in through Google SSO when prompted. The script saves your session cookies so future runs work without manual login.

cd C:\Users\mberning\projects\eusd\thrillshare-checker
venv\Scripts\activate
python checker.py --login

A Chromium window will open. You'll see a prompt:

=== LOGIN REQUIRED ===
Please log in using Google in the browser window.
Take your time — the script will wait.

Complete the Google SSO login. Once done, the script saves your cookies and proceeds to check all schools.

2 Run an Automated Check

After the initial login, run without flags for a headless check. If pending items are found, an email is sent automatically.

python checker.py

3 Run Without Sending Email

To see results in the terminal without sending email (useful for testing):

python checker.py --no-email

4 Scheduling (Optional)

The tool runs automatically via Windows Task Scheduler using batch file wrappers. The wrappers run silently, write output to log files, and support S4U logon (run without an active user session — machine can be fully logged off).

Two scheduled tasks are configured:

Task NameScheduleWrapperLog File
ThrillshareCheckerWeekdays (Mon–Fri) at 4:00 AM and 7:20 AM + hourly 6 AM – 4 PMrun_hidden.batlog.txt
Thrillshare Smore ScanWeekdays (Mon–Fri) at 3:00 AM and 6:50 AMrun_smore_scan.batsmore_scan_log.txt

Both tasks have StartWhenAvailable and WakeToRun enabled. StartWhenAvailable fires a single catch-up run if the machine was off or rebooting at the scheduled time. WakeToRun wakes the machine from its idle sleep (AC standby is 2 hours) so early-morning runs are not silently skipped. Each task runs cleanup_stale.ps1 first to kill any Chrome or Python processes left over from a previous hung run. Output is written live to the log file as each school is checked.

Step-by-Step: Smore Link Scanner

The Smore scanner walks every school the account can see and extracts any app.smore.com URLs from messages. There is no sender filter — it picks up principals, office managers, and district staff. For every row in the analytics table the scanner reads the message text directly (7-day / 168h lookback). For Email rows sent in the last 48h it also opens the "View Message" popup to expand the full body.

After the initial discovery pass, the scanner runs a follow-up pass: it re-fetches every previously discovered Smore URL whose first snapshot is approximately 24h or 48h old, captures the current page text, and emails you a unified diff if the content changed. If a URL is no longer reachable, a separate "unreachable" email is sent instead.

Deduplication: Each unique Smore URL is emailed only once on first detection (tracked in session/seen_smore.json). Follow-up emails are sent separately when the page changes or becomes unreachable. Entries auto-expire after 30 days.

When a new Smore URL is first detected, the scanner also automatically queues a background ADA accessibility scan via the local smore-ada-checker dashboard. The scan runs in the background (~1-2 min) and requires no manual action — check the dashboard when ready to review. No additional email is sent from this path. This behavior requires the smore-ada-checker project to be installed at SMORE_CHECKER_DIR (see environment variables); if the directory is missing, a warning is printed and the scan is skipped.

1 Run a Manual Smore Scan

cd C:\Users\mberning\projects\eusd\thrillshare-checker
venv\Scripts\activate
python checker.py --smore-scan --no-email

2 Run With Email Notification

Drop the --no-email flag. An email is always sent — subject line indicates whether links were found. Follow-up diff/unreachable emails are sent in the same run if any 24h/48h checks are due.

python checker.py --smore-scan

3 Skip Follow-Up Checks

To run only the initial discovery pass (no re-fetch of previously seen URLs):

python checker.py --smore-scan --no-followup

5 Adjusting Coverage

The scanner uses whatever schools the dropdown returns from discover_schools — no allowlist to maintain. To change the time window, edit LOOKBACK_HOURS (default: 168) or POPUP_LOOKBACK_HOURS (default: 48) at the top of smore_scanner.py. The scheduled task fires on weekdays (Mon–Fri).

CLI Reference

FlagDefaultDescription
--loginOffForce a visible browser login. Use for first-time setup or when your session expires.
--no-emailOffPrint results to the terminal only. No email is sent regardless of findings.
--smore-scanOffScan every school's analytics for Smore links instead of checking approvals. Row-text lookback: 168h (configurable via LOOKBACK_HOURS); popup expansion: 48h (POPUP_LOOKBACK_HOURS).
--no-followupOffSkip the 24h/48h follow-up re-check pass when running --smore-scan. Only the initial discovery pass runs; no diff or unreachable emails are sent.

With no flags, the tool runs headless and always sends an email. Use --no-email for local testing without sending.

Troubleshooting

ErrorCauseFix
Session expired. Run with --login to re-authenticate. Saved cookies have expired or become invalid Run python checker.py --login and complete the Google SSO login
No schools discovered. The organization dropdown wasn't found within 30s. Usually a session or network issue; the checker already waits up to 30s for the SPA OAuth handshake. Run with --login to see the browser. Verify you can see the school switcher dropdown. Check that your session cookies are valid (python checker.py --login to re-authenticate). If the Thrillshare UI changed, selectors in checker.py may need updating.
Could not find MEDIA nav link! The MEDIA navigation link wasn't found after logging in Run with --login to check whether you land on the accounts page. Ensure your Google account has Thrillshare media access.
Failed to send email: ... SMTP relay connection failed Check your .env settings. Verify SMTP_HOST (default: mail.eusd.org) and SMTP_PORT (default: 25) are correct. Ensure the machine can reach the mail server on the network.
Timed out waiting for login. The 2-minute login timeout was reached before SSO completed Run with --login again and complete the Google login more quickly. The script waits up to 2 minutes.
Scheduled task runs but no email and no report The batch wrapper or paths may be misconfigured, or a stale process is blocking the task Check log.txt and smore_scan_log.txt — output is written live during the run, so the file should grow as each school is checked. If the file is empty and Task Scheduler shows Status=Running but no python.exe is visible, a stale cmd or Chrome process may be blocking it; run cleanup_stale.ps1 manually (as Administrator) to reap orphans, then re-trigger the task. Also verify paths in run_hidden.bat / run_smore_scan.bat match your project location.
Error checking CMS: ... The CMS pages section failed to load or the table structure changed Run manually with --no-email to see the full error. The CMS check navigates directly to thrillshare.com/cms/v2/pages — verify that URL works in a normal browser.

Key Files

  • .env — SMTP host, port, sender/recipient email, SMORE_CHECKER_DIR (set to empty string to disable ADA handoff)
  • session\cookies.json — saved login session
  • session\seen_smore.json — dedupe cache + follow-up state for Smore links
  • session\smore_snapshots\ — plain-text page snapshots used for follow-up diffs
  • smore_scanner.py — time window settings (LOOKBACK_HOURS, POPUP_LOOKBACK_HOURS)
  • reports\ — date-organized report history (auto-cleaned after 3 days)
  • log.txt / smore_scan_log.txt — output from scheduled runs
