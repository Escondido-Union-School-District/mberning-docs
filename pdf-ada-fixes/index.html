What This Tool Does

This tool automatically fixes four common PDF accessibility issues required for ADA/Section 508 compliance:

#FixWhat It Does
1Figure Alt TextUses Google Gemini AI to generate descriptive alt text for all images
2Document LanguageSets the document language to English (en-US)
3Document TitleExtracts the first <H1> heading and sets it as the document title
4Table HeadersMarks table header cells so screen readers can navigate tables
Requirement: Input PDFs must be tagged PDFs (exported from Word with "Create bookmarks using Headings" or similar). Untagged/scanned PDFs will not work.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.10+ must be installed
  2. Project folder: C:\Users\mberning\projects\eusd\pdf-ada-fixes
  3. Virtual environment with dependencies is already created in venv\
  4. Google API key is saved in the .env file

If Setting Up on a New Machine

# 1. Clone or copy the project folder

# 2. Open a terminal in the project folder and create the virtual environment:
python -m venv venv

# 3. Activate it:
#    PowerShell:
.\venv\Scripts\Activate.ps1
#    Git Bash / Terminal:
source venv/Scripts/activate

# 4. Install dependencies:
pip install -r requirements.txt

# 5. Create the .env file with your Google API key:
#    Copy .env.example to .env, then edit .env and replace the placeholder:
GOOGLE_API_KEY=your-actual-api-key-here
Getting a Google API Key: Go to aistudio.google.com, sign in, click "Get API key", and create a key. The free tier allows 15 requests per minute which is sufficient for this tool.

How to Run the Tool

1 Place PDF files in the input\ folder

Copy the PDF files you want to fix into:

C:\Users\mberning\projects\eusd\pdf-ada-fixes\input\
Important: Make sure the input PDFs are not open in Acrobat or any other program. Open files will be locked and the tool won't be able to move them to processed\ when finished.

2 Open a terminal

Open Git Bash, PowerShell, or Windows Terminal and navigate to the project:

cd C:\Users\mberning\projects\eusd\pdf-ada-fixes

3 Activate the virtual environment

# PowerShell:
.\venv\Scripts\Activate.ps1

# Git Bash:
source venv/Scripts/activate

You should see (venv) appear at the start of your prompt.

4 Run the tool

Process all PDFs in the input folder (recommended):

python remediate.py input/

Or process a single file:

python remediate.py "input/YourFile.pdf"

Example with header option:

python remediate.py input/ --headers row

5 Collect your output

Remediated files are saved to the output\ folder with _remediated appended to the filename. The original input file is automatically moved to the processed\ folder.

Batch mode: When you pass a folder instead of a file, the tool processes every .pdf in that folder and prints a summary at the end showing how many succeeded or failed.

The --headers Option

This controls how table headers are marked. Choose based on your table layout:

OptionUse WhenWhat It Does
--headers row First row contains column headings (most common) Marks first row cells as <TH Scope=Column>
--headers column First column contains row headings Marks first cell in each row as <TH Scope=Row>
--headers both Both first row and first column are headings Applies both row and column header fixes
--headers none Tables don't have headers or are already correct Skips table header fixing entirely
Default: If you don't specify --headers, it defaults to row (first row = column headers). This is correct for most EUSD implementation plan templates.

Processing Multiple Files

The easiest way to process all PDFs at once is to pass the folder path:

python remediate.py input/

This processes every .pdf file in the folder and prints a batch summary when finished.

Folder Structure

pdf-ada-fixes\
  input\          <-- Put your PDFs here
  output\         <-- Remediated PDFs appear here (*_remediated.pdf)
  processed\      <-- Original PDFs are moved here after processing

  .env            <-- Your Google API key (don't share!)

Troubleshooting

ErrorCauseFix
GOOGLE_API_KEY not set Missing or empty .env file Create/edit .env with your API key
PDF is not tagged The PDF wasn't exported as a tagged PDF Re-export from Word with accessibility tags enabled
file not found Wrong path or filename Check the filename and use quotes around paths with spaces
[SKIP] Figure N: could not extract image Image format not extractable Tool uses a screenshot fallback; if both fail, add alt text manually in Acrobat
ResourceExhausted / rate limit Too many Gemini API calls too fast Wait a minute and try again. The tool has built-in delays.
(venv) not showing Virtual environment not activated Run the activate command from Step 3
Could not move ... to processed/ Input PDF is still open in Acrobat or another program Close the PDF in Acrobat before running the tool, or close it after and move it to processed\ manually
Tip: Close input PDFs in Acrobat before running the tool. If a PDF is open in another program, the tool will still create the remediated file in output\, but it won't be able to move the original to processed\.
