What This Tool Does

The EUSD Link Checker crawls all 27 Escondido Union School District websites, discovers every hyperlink on each page, and checks whether those links still work. It produces reports listing broken, unreachable, and anomalous links so the web team can fix them.

Sites covered: the main district site plus all elementary, middle, and specialty school sites hosted under *.eusd.org.

Reports generated:

  • Internal issues — broken links within EUSD domains
  • External issues — broken links to outside sites
  • Anomalies — likely false positives (Google, Zoom, LinkedIn, etc. block bots)
  • Link matches — pages linking to a specific target URL (optional)
  • Link inventory — catalog of all internal links discovered

Prerequisites

One-Time Setup (already done on your machine)

  1. Node.js 18+ installed (download from https://nodejs.org)
  2. npm (included with Node.js)
  3. Project folder: C:\Users\mberning\projects\eusd\eusd-linkcheck
  4. Dependencies installed via npm install
  5. Internet access — the tool crawls live EUSD sites
  6. ~2 GB free RAM — Playwright runs headless browsers

If Setting Up on a New Machine

# 1. Clone or copy the project folder

# 2. Open a terminal in the project folder and install dependencies:
npm install

# Playwright will also download browser binaries (~150 MB) on first install.
# If browsers are missing later, run:
npx playwright install

Step-by-Step: Running the Link Checker

Open a terminal in the project folder (C:\Users\mberning\projects\eusd\eusd-linkcheck).

1 Quick run (default — 300 pages)

npm start

Crawls up to 300 pages, checks all discovered links, and writes reports to output\YYYY\MM\.

2 Full site crawl

npm run start:full

Crawls every reachable page across all 27 EUSD sites. This can take a long time depending on site size and network speed.

3 Find pages linking to a specific URL

TARGET_URL="https://example.com/page" npm start

Generates an extra link-matches report showing which EUSD pages link to the target URL.

4 Send reports via email

EMAIL_USER="you@eusd.org" EMAIL_PASS="xxxx-xxxx-xxxx-xxxx" npm run email

Emails the most recent reports as attachments. Requires a Google App Password (see the Email Setup section below).

5 Full crawl + email in one command

EMAIL_USER="you@eusd.org" EMAIL_PASS="xxxx-xxxx-xxxx-xxxx" npm run report

Link categories

Each issue is classified with one of these categories:

  • BROKEN — HTTP status 400 or higher, or status 0 (server refused connection)
  • UNREACHABLE — request timed out or a network error occurred
  • NOT_CHECKED — link was skipped (e.g., external checks disabled)

HTML reports

Open any .html report file in a browser to get an interactive view with:

  • A search box for filtering across all fields
  • Category filter buttons (All / Broken / Unreachable / Not Checked)
  • Color-coded cards with clickable links
  • Summary counts at the top

CSV reports

CSV files can be opened in Excel or Google Sheets. Columns:

category, scope, page_title, source_url, link_text, link_href, link_resolved, checked_url, status_code, final_url, reason, error
Partial runs: If a crawl is interrupted, output files will have -partial appended to the filename. These contain valid data but represent an incomplete crawl.

Caching

The tool uses a multi-level caching strategy to avoid re-checking links that were recently confirmed working:

  1. In-memory LRU cache (up to 10,000 entries) — avoids duplicate checks within a single run
  2. Persistent disk cache (cache\status-cache.json) — remembers URLs that returned 200 OK for up to 7 days

Only successful (200) responses are cached on disk. Errors are always re-checked.

# Disable disk cache for one run:
NO_CACHE=1 npm start

# Or delete the cache file entirely:
rm cache/status-cache.json

Email Setup (Google App Password)

The email feature uses Gmail SMTP. You need a Google App Password, not your regular password.

1 Open App Passwords

Go to https://myaccount.google.com/apppasswords (requires 2FA enabled on your Google account).

2 Create a new app password

Name it something like "EUSD Link Checker".

3 Copy the password

Copy the 16-character password (formatted like abcd efgh ijkl mnop).

4 Use it when sending email

EMAIL_USER="you@eusd.org" EMAIL_PASS="abcdefghijklmnop" npm run email
Security: Never commit your App Password to version control. Set it as an environment variable or pass it on the command line.

Troubleshooting

ErrorCauseFix
Missing browser executables Playwright browsers not installed Run npx playwright install
No reports found. Run the link checker first. Running npm run email before any crawl has completed Run npm start first to generate reports, then npm run email
EMAIL_USER and EMAIL_PASS environment variables are required Attempting to send email without credentials Set both environment variables (see Email Setup section above)
Many links showing as UNREACHABLE Network issues, firewall blocking, or sites rate-limiting the checker Increase timeout: LINK_TIMEOUT_MS=20000 npm start
Crawl stops at 300 pages Default page limit is 300 Use npm run start:full for a complete crawl
Rate limiting detected (backing off...) More than 30% of recent requests are failing; tool is auto-slowing No action needed — it resumes normal speed once the error rate drops
High memory usage Too many concurrent browsers/checks during large crawls Reduce concurrency: MAX_CONCURRENCY=2 LINK_CHECK_CONCURRENCY=10 npm run start:full
Anomalies report has many entries Google, Zoom, LinkedIn, Facebook, Twitter, Instagram block bots These links are almost always fine — the tool separates them so they don't pollute real reports
