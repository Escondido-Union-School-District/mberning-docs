What This Tool Does

Audits all EUSD district webpages for ADA/WCAG 2.1 Level AA compliance as required by the DOJ Title II web accessibility rule (deadline: April 26, 2027). The tool runs 15 automated and semi-automated checks on every page, tracks remediation progress, and generates reports.

FeatureWhat It Does
Automated auditRuns axe-core + custom checks against rendered pages via Playwright
Manual review queueFlags items that need human verification with copy-paste commands
DashboardHTML progress dashboard showing status by check and by site
Daily nudges"Hey Mark" emails with today's batch, how to start, and schedule tracking
Low-hanging fruitIdentifies the most impactful fixes across all pages
Compliance deadline: April 26, 2027 for entities under 50,000 population. Federal penalties up to $75,000 first offense.

Prerequisites

One-Time Setup (already done on your machine)

  1. Node.js 18+ installed
  2. Project folder: C:\Users\mberning\projects\eusd\doj-webrule-audit
  3. Dependencies installed via npm
  4. Playwright Chromium installed (npx playwright install chromium)

If Setting Up on a New Machine

# 1. Clone the repository
gh repo clone Escondido-Union-School-District/doj-webrule-audit

# 2. Install dependencies
cd doj-webrule-audit
npm install
npx playwright install chromium

# 3. Import the Excel data
npm run import

# 4. (Optional) Configure email notifications
# Copy .env.example to .env and fill in EMAIL_USER, EMAIL_PASS, EMAIL_TO

Step-by-Step

1 Start your day

Open a terminal and type:

dwa

This launches Claude Code in the project directory. Then run:

npm run today

This shows where you left off, today's batch, and copy-paste commands for every item.

2 Run automated scans (if needed)

If there are unscanned pages, the "today" output will tell you. Run:

# Scan 50 unscanned pages
npm run audit -- --new --limit 50

# Scan a specific site
npm run audit -- --site pioneer --limit 20

# Scan a single page
npm run audit -- --url https://www.eusd.org

3 Do manual reviews

The "today" output lists items needing manual review. For each one:

  1. Open the URL in your browser
  2. Perform the check described (e.g., test keyboard navigation)
  3. Record the result using the provided command:
# Pass a check
npm run review -- 142 1 pass "Keyboard nav works on all menus"

# Fail a check
npm run review -- 142 1 fail "Dropdown menu traps keyboard focus"

4 Batch-apply template-level results

Since all EUSD sites use the same Apptegy template, some checks only need verification once:

# All pages pass skip links (verified on template)
npm run review:batch -- --check 3 pass "Skip link verified on template"

# All pioneer pages pass magnification
npm run review:batch -- --check 13 --site pioneer pass "Zoom to 400% OK"

5 Check progress

# Quick status
npm run status

# Full HTML dashboard
npm run dashboard

# Low-hanging fruit report
npm run quickwins

Understanding the Output

Each page gets 15 checks. Each check produces one of these statuses:

StatusMeaning
passCheck passed automated verification
failCheck found accessibility issues
needs-reviewAutomation couldn't determine pass/fail — needs manual check
n/aCheck not applicable (e.g., no tables on page)
errorPage failed to load or check errored

The dashboard at output\dashboard.html shows aggregate results by check, by site, top issues, and daily progress.

Troubleshooting

ErrorCauseFix
Page load failed: Timeout Page took too long to render Retry, or increase PLAYWRIGHT_TIMEOUT in .env
axe-core failed Page structure prevented axe from running Try auditing the page again; if persistent, do manual audit
Email not configured Missing .env email settings Copy .env.example to .env and fill in EMAIL_USER, EMAIL_PASS, EMAIL_TO
No pages to audit Database is empty Run npm run import first to load Excel data
