What This Tool Does

Audits all EUSD district webpages for ADA/WCAG 2.1 Level AA compliance as required by the DOJ Title II web accessibility rule (deadline: April 26, 2027). The tool runs 15 automated and semi-automated checks on every page, tracks remediation progress, and generates reports.

FeatureWhat It Does
Automated auditRuns axe-core + custom checks against rendered pages via Playwright
Review UIInteractive web tool for reviewing pages — click to toggle pass/fail, add notes, filter by status/site/check
Linked files pageReview linked PDFs and documents (check 14) with file-level status tracking
Progress dashboardReal-time stats in Review UI header — pages passed, reviewed, daily goals, weekly progress
Team dashboardRead-only GitHub Pages dashboard for sharing progress with the team
Daily nudges"Hey Mark" emails with today's batch, how to start, and schedule tracking
Low-hanging fruitIdentifies the most impactful fixes across all pages
Compliance deadline: April 26, 2027 for entities under 50,000 population. Federal penalties up to $75,000 first offense.

Prerequisites

One-Time Setup (already done on your machine)

  1. Node.js 18+ installed
  2. Project folder: C:\Users\mberning\projects\eusd\doj-webrule-audit
  3. Dependencies installed via npm
  4. Playwright Chromium installed (npx playwright install chromium)

Step-by-Step

1 Start your day

Open a terminal and type:

dwa

This launches Claude Code in the project directory. The Review UI auto-starts on login (Windows scheduled task), so it should already be running at http://localhost:3000. If not:

npm run review-ui

2 Review pages in the Review UI

The Review UI is the primary workflow tool. Open http://localhost:3000 in your browser.

  • Dashboard header shows Fully Passed, Reviewed w/ Failures, Need Review, Today's progress vs daily goal, and weekly/monthly counts
  • Check progress bar shows remaining count per check; fully-passed checks auto-hide
  • Filter bar lets you filter by status, site, check number, or search by page name
  • Page grid shows each page with all 15 checks in a compact table

Reviewing a page

  • Click P/F/? cells to cycle: unreviewed → pass → fail → unreviewed
  • Click note cells to open a popup for adding notes
  • Pass All button marks all unreviewed checks as pass (with Undo)
  • Skip button removes unpublished pages from review
  • Check 14 links to the Linked Files page for pages that have linked documents

3 Review linked files

Click Linked Files → in the header or check 14's link to go to the files page. This shows all linked PDFs/documents grouped by page, with file-level pass/fail tracking that rolls up to check 14.

4 Run automated scans (if needed)

If there are unscanned pages:

# Scan 50 unscanned pages
npm run audit -- --new --limit 50

# Scan a specific site
npm run audit -- --site pioneer --limit 20

# Scan a single page
npm run audit -- --url https://www.eusd.org

5 Check progress

# Quick CLI status
npm run status

# Low-hanging fruit report
npm run quickwins

# Update team dashboard (GitHub Pages)
npm run export-site

6 CLI-based reviews (alternative to Review UI)

You can still record reviews from the command line if preferred:

# Pass a check
npm run review -- 142 1 pass "Keyboard nav works on all menus"

# Fail a check
npm run review -- 142 1 fail "Dropdown menu traps keyboard focus"

# Batch-apply template-level results
npm run review:batch -- --check 3 pass "Skip link verified on template"

Understanding the Output

Each page gets 15 checks. Each check produces one of these statuses:

StatusIn Review UIMeaning
passGreen PCheck passed — automated or manually verified
failRed FCheck found accessibility issues
unreviewedYellow ?Not yet reviewed — click to set pass or fail

The Review UI dashboard header shows aggregate counts. A page is "Fully Passed" when all 15 checks are pass. "Reviewed w/ Failures" means all 15 checks have been reviewed but at least one failed.

Troubleshooting

ErrorCauseFix
Page load failed: Timeout Page took too long to render Retry, or increase PLAYWRIGHT_TIMEOUT in .env
axe-core failed Page structure prevented axe from running Try auditing the page again; if persistent, do manual audit
Review UI not loading Server not running Run npm run review-ui or check if port 3000 is in use
Email not configured Missing .env email settings Copy .env.example to .env and fill in EMAIL_USER, EMAIL_PASS, EMAIL_TO
No pages to audit Database is empty Run npm run import first to load Excel data
