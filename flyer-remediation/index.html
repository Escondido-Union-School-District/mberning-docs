PDF Flyer Remediation — User Guide

User Guide — Escondido Union School District

Last updated: 2026-04-16 — v2 text-first pipeline, QR codes, clickable links, PDF bookmarks, text quality checks

What Is It?

This tool takes PDF flyers exported from Canva (or any flat PDF) and rebuilds them so screen readers can read them correctly. It is for the EUSD Communications or Technology team to process event flyers before posting them publicly. No design experience is needed — just drop in a PDF and run one command.

What This Tool Does

remediate.py performs a full PDF/UA-1 accessibility remediation on each input flyer. It extracts all text directly from the PDF, then asks an AI vision model to classify each piece of text into semantic roles (heading, paragraph, figure, etc.) by comparing it to the page image. A fresh, standards-compliant tag tree is then written from scratch — the original (often broken) Canva tag tree is completely discarded.

StepWhat Happens
Extract text All text lines are pulled directly from the PDF and de-duplicated (Canva exports each line twice for drop-shadow effects).
AI classification The extracted text list and a full-page image are sent to the AI together. The AI assigns each text line a role (H1/H2/H3/P/Link/Artifact), a reading order, and writes alt text for images — in the document's language.
Quality check An automated check scans for missing spaces, single-character fragments, and encoding errors. A second Vision pass cross-checks the text against the page image.
Tag & save The original content stream is preserved but wrapped as artifact content. An invisible accessible text layer is appended at the original text positions. A fresh tag tree, PDF bookmarks, clickable link annotations, and metadata (title, language, PDF/UA marker) are written.

What Gets Tagged Automatically

  • Headings (H1–H3): Proper hierarchy enforced — no skipped levels. H1 is always the first element a screen reader encounters.
  • Images: Each photo or graphic gets an alt-text description written by the AI in the document's language (English or Spanish).
  • URLs and links: Any text containing a URL is tagged as a clickable Link — screen readers announce it as a navigable link, not plain text.
  • QR codes: If a QR code image is detected, the AI decodes the URL it encodes and includes it in the alt text so screen reader users get equivalent access.
  • PDF bookmarks: All headings are added as PDF bookmarks so users can jump between sections in Acrobat's Bookmarks panel.
  • Language: The document language (e.g., en-US, es) is auto-detected from the visible text and set on the document. Spanish flyers get Spanish alt text.
Note: Every output is a draft. You must review the result in Adobe Acrobat Pro and run PAC 2024 before publishing. See the Review Workflow section below.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.10+ installed
  2. Project folder: ~/projects/eusd/flyer-remediation/ with a virtual environment in venv/
  3. API key configured in ~/projects/eusd/flyer-remediation/.env
  4. Dependencies installed (pikepdf, pymupdf, anthropic, google-genai, Pillow, python-dotenv)

If Setting Up on a New Machine

# 1. Copy the project folder to ~/projects/eusd/flyer-remediation/

# 2. Open a terminal in the project folder:
cd ~/projects/eusd/flyer-remediation

# 3. Create and activate a virtual environment:
python -m venv venv
source venv/Scripts/activate      # Git Bash / MSYS2
# OR: venv\Scripts\activate        # PowerShell / cmd

# 4. Install dependencies:
pip install -r requirements.txt

# 5. Copy the env template and fill in your API key:
cp .env.example .env
# Edit .env and set GOOGLE_API_KEY or ANTHROPIC_API_KEY
API Key: The tool requires either a Google Gemini API key (GOOGLE_API_KEY) or an Anthropic Claude API key (ANTHROPIC_API_KEY). The VISION_PROVIDER setting in .env controls which one is used. Contact the project owner to obtain the correct key.

Step-by-Step

Run this whenever you have new flyers to make accessible.

1 Drop your PDF(s) into the input folder

Copy or move the flyer PDF(s) you want to process into:

~/projects/eusd/flyer-remediation/input/

You can drop in one file or an entire batch of files at once.

2 Open a terminal and activate the virtual environment

cd ~/projects/eusd/flyer-remediation
source venv/Scripts/activate

Your prompt should change to show (venv) at the start.

3 Run the tool

Process a single file:

python remediate.py input/MyFlyer.pdf

Process all PDFs in the input folder at once:

python remediate.py input/

4 Wait for it to finish

A typical 1-page flyer takes 15–30 seconds. You will see progress printed for each page. A successful run ends with:

Saved: output/MyFlyer_uaremediated.pdf

============================================================
Batch complete: 1 succeeded, 0 failed
============================================================

5 Review the output in Acrobat Pro

Open ~/projects/eusd/flyer-remediation/output/MyFlyer_uaremediated.pdf in Adobe Acrobat Pro and follow the Review Workflow below.

6 Validate in PAC 2024

Drag the output PDF into PAC 2024. Expected result: zero errors on Matterhorn checkpoints 01–31.

Review Workflow

Every output is a draft. Complete these checks in Acrobat Pro before publishing.

  1. Tags panel: Go to View → Tags panel. Expand <Document>. Click each tag — a pink border must appear around the real visual element on the page. This is the acceptance criterion.
  2. Accessibility Check: Run Tools → Accessibility → Accessibility Check. Only the two manual checkpoints ("Logical Reading Order" and "Color Contrast") should require human review. All machine checks must pass.
  3. Reading Order panel: Confirm the numbered order matches how you would naturally read the flyer top-to-bottom.
  4. Document Properties: Go to File → Properties. Verify: Title is present, Language is en-US, and "Tagged PDF: Yes" is shown.
  5. PAC 2024: Drag the PDF into PAC 2024. Expect zero errors on Matterhorn checkpoints 01–31.
Most common manual fix needed: Reading order on dense layouts (hero image + sidebar + callout + footer) is where AI errors appear most. If the pink border lands on the wrong element, drag the tag to its correct position in the Tags panel.

CLI Reference

FlagDefaultDescription
input_path required Path to a single PDF file or a folder of PDFs to process.
--dpi 200 Resolution (dots per inch) used when rendering pages for the AI. Higher values improve accuracy on small text but increase cost and time.
--force-raster off Skip live-text matching entirely and OCR every region. Use this if the PDF has no selectable text at all (fully rasterized).
--dry-run off Analyze and print the tagging plan without saving any files. Use to preview what will happen before committing.
--verify off After saving, re-open the PDF and confirm every tag in the structure tree is properly linked to a page content stream marker. Adds a few seconds per file.
--verbose off Print full AI responses and per-tag diagnostics. Use when troubleshooting a bad output — output can be very long.

Common invocations:

# Process one file
python remediate.py input/SummerFlyer.pdf

# Process a whole batch
python remediate.py input/

# Preview without saving
python remediate.py input/SummerFlyer.pdf --dry-run

# Extra quality check after saving
python remediate.py input/SummerFlyer.pdf --verify

# Troubleshoot a bad result
python remediate.py input/SummerFlyer.pdf --verbose

Understanding the Output

For each input PDF named MyFlyer.pdf, the tool creates output/MyFlyer_uaremediated.pdf. The original is moved to processed/MyFlyer.pdf so input/ stays clean.

Console output during a typical run:

Vision provider: gemini

============================================================
Processing (v2): SummerFlyer.pdf
============================================================

  Page 1/1: extracting text (dedup)...
  Page 1: text_lines=38 images=6
  Page 1: vision classify call...
  Page 1: lang='en-US' elements=12
  Page 1: analyzing content stream (images only)...
  Text check: OK

  Saved: output/SummerFlyer_uaremediated.pdf

============================================================
Batch complete: 1 succeeded, 0 failed
============================================================

Key lines to look for:

  • text_lines=N images=N — how many text lines and images were extracted from the PDF. If text_lines=0, the PDF is fully rasterized and the AI will tag a placeholder.
  • lang='xx' elements=N — language detected and number of semantic elements the AI classified.
  • Text check: OK — automated quality check passed. If issues are found, details are printed before this line.
Note: If you see [WARN] vision_classify returned None, the AI failed to classify the page after two attempts. The page will be tagged as a single paragraph. Run again with --verbose to see the raw AI response, or try increasing --dpi for better image quality.

Troubleshooting

Error / SymptomCauseFix
GOOGLE_API_KEY not set or ANTHROPIC_API_KEY not set The .env file is missing or the key variable is blank. Open .env and set the correct API key for your VISION_PROVIDER. See .env.example for the required variable names.
Error: path not found The file or folder path you typed does not exist. Check spelling and confirm the file is in the input/ folder. Use ls input/ to list what is there.
No PDFs in [path] The folder exists but contains no .pdf files. Confirm the files you dropped in have a .pdf extension (not .PDF capitalized on some systems).
Error: PDF is encrypted The input PDF is password-protected. Open the PDF in Acrobat and remove the password (File → Properties → Security), then re-save and try again.
Error: could not open The PDF is corrupted or not a valid PDF file. Re-export from Canva or the original source and try again. If it opens fine in Acrobat, try File → Save As to re-save a clean copy.
[WARN] Could not move [filename] (locked) Another application (Acrobat, antivirus) has the original file open. Close the file in any other program and the warning is harmless — the output was still saved. Move the original manually to processed/ when ready.
[WARN] vision_classify returned None The AI failed to return a valid classification after two attempts (often a JSON truncation on dense pages). The page is tagged as a single paragraph. Re-run with --verbose to see the raw AI response. Try --dpi 300 for a higher-quality image, or split a dense bilingual flyer into separate files.
Verify shows orphan=N FAIL Some structure tree tags are not linked to a page content stream marker. Indicates an internal tool error. Run again with --verbose and report the full output. As a workaround, open in Acrobat and manually fix or delete the orphaned tags.
Pink border lands on wrong element in Acrobat AI misclassified the reading order or role on a complex layout (common on multi-column or sidebar designs). Drag the tag to its correct position in the Acrobat Tags panel. Use --verbose to inspect the AI classification and understand what it saw.
Screen reader reads text out of order Multi-column or sidebar layout confused the AI's reading-order assignment. Open the Reading Order panel in Acrobat and drag items to the correct sequence. This is the most common manual fix needed on complex flyers.
Text quality issues printed during run Automated check found potential word-boundary or encoding problems (single characters, missing spaces). Check the flagged elements in the output PDF. If incorrect, the AI's Vision cross-check may have already corrected minor issues. For persistent problems, run with --verbose for full detail.
Virtual environment not activated (python: command not found or wrong version) You skipped the source venv/Scripts/activate step. Run source venv/Scripts/activate from the project folder first, then re-run the command.
