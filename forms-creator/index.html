What Is Forms Creator?

Forms Creator is a tool that lets school office staff build and publish bilingual (English and Spanish) web forms without writing any code. You start from a pre-approved template — such as an allergy waiver or permission slip — rename it for your school or program, edit the labels right on screen, and click Publish. The tool generates a complete web form that parents can fill out online, along with a Google Apps Script that saves every submission directly into a Google Sheet.

What This Tool Does

Forms Creator runs as a small local web application. You open it in your browser, do your editing, and publish. The published form is a folder of static files you can drop onto GitHub Pages or your district web server. Submissions come back to a Google Sheet of your choice.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.10 or later must be installed
  2. Project folder: ~/projects/eusd/forms-creator/
  3. Python dependencies installed via pip (see setup steps below if on a new machine)

If Setting Up on a New Machine

# Open PowerShell and navigate to the project folder
cd C:\Users\YourName\projects\eusd\forms-creator

# Create a virtual environment (one time only)
python -m venv venv

# Activate it
.\venv\Scripts\Activate.ps1

# Install dependencies (one time only)
pip install -r requirements.txt
If Activate.ps1 is blocked: Run PowerShell as Administrator and execute Set-ExecutionPolicy RemoteSigned, then try again.

Step-by-Step

Follow these steps each time you create a new form, from cloning a template to a live published form.

1 Start Forms Creator

Open a PowerShell terminal in the project folder and run:

.\venv\Scripts\Activate.ps1
python app.py

You should see output like:

 * Running on http://127.0.0.1:5000
 * Debug mode: on

Open your browser and go to http://localhost:5000. Leave the terminal open while you work — closing it stops the server.

2 Clone a Template

The dashboard shows two panels: My Forms (forms you have already created) and Templates (the pre-approved starting points).

  1. In the Templates panel, find the template closest to your form (for example, allergy-waiver or permission-slip).
  2. Click the template name. A small form appears asking for a New Form ID.
  3. Type a short, lowercase ID using only letters, numbers, and hyphens (for example: 2026-allergy-waiver or spring-field-trip). No spaces.
  4. Click Create. You land on the edit page for your new form.

3 Edit the Form

The edit page shows the form's title, description, and all sections with their fields.

  1. Rename the form title: Click on the title text. An edit popover appears with two boxes — one for English and one for Spanish. Type your new title in both, then click Save.
  2. Edit a field label: Click on any field label. The same EN/ES popover appears. Update both languages and save.
  3. Add a field: At the bottom of any section, click + Add Field. Choose a field type (text, date, checkbox, radio, dropdown), enter the ID and labels, and click Add.
  4. Delete a field: Click the trash icon to the right of any field, then confirm.
  5. Reorder fields: Drag the handle (↕) on the left of any field row up or down to change its position in the section.
Field types: text (single line), text_multiline (paragraph), date (date picker), checkbox (yes/no tick), radio (pick one from a list), dropdown (pick one from a menu). Radio and dropdown fields require at least one option.

4 Configure Outputs

Click Outputs in the edit page's sidebar (or the Outputs tab at the top). This panel controls what gets generated when you publish.

SettingWhat It Does
Live web form — Enabled Generates the public HTML form. Leave this checked.
Publish path The folder name for the export (e.g. 2026-allergy-waiver). Use lowercase letters, digits, and hyphens only.
Spreadsheet — Enabled Generates an Apps Script to capture submissions in a Google Sheet. Check this if you want responses saved automatically.
Sheet ID The ID of the Google Sheet where submissions will be saved (see step 6 for how to find it).
Sheet ID format: Open the Google Sheet in your browser. The Sheet ID is the long string of letters and numbers in the URL between /d/ and /edit.
Example URL: https://docs.google.com/spreadsheets/d/1BxiMVs0XRA5nFMdKvBdBZjgmUUqptlbs74OgVE2upms/edit

5 Preview the Form

Before publishing, click Preview to see the live form exactly as parents will see it. The preview is for review only — it does not save submissions. Use the language toggle button at the top of the preview to confirm both English and Spanish look correct.

6 Publish

When everything looks right, click Publish on the edit page. Forms Creator generates the export folder at:

~/projects/eusd/forms-creator/data/exports/<publish-path>\

Inside you will find:

index.htmlThe bilingual web form parents fill out
form.cssStyling for the web form (do not edit)
form.jsLanguage switching and submission logic (do not edit)

7 Set Up the Apps Script

The Apps Script is Google's server-side code that catches each form submission, saves a row in your Sheet, and stores a PDF copy in Google Drive.

  1. Open your Google Sheet in a browser.
  2. Click Extensions → Apps Script. A code editor opens in a new tab.
  3. Select all the existing placeholder code (Ctrl+A) and delete it.
  4. Open apps_script.gs from your export folder in a text editor (Notepad works). Select all, copy.
  5. Paste into the Apps Script editor and click the floppy-disk Save icon (or press Ctrl+S).
  6. Click Deploy → New deployment.
  7. Next to "Select type," click the gear icon and choose Web app.
  8. Set Execute as to Me and Who has access to Anyone.
  9. Click Deploy. Google will ask you to authorize the script — click Authorize access and sign in with your district Google account.
  10. Copy the Web app URL that appears (it looks like https://script.google.com/macros/s/AKfyc.../exec).
Authorization warning: Google may show a screen saying "Google hasn't verified this app." Click Advanced → Go to [your project name] (unsafe). This is expected for district-internal scripts and is safe to proceed.

8 Wire the Submission URL

After deploying the Apps Script, you need to tell the web form where to send submissions.

  1. Open the config.json file from your export folder in a text editor.
  2. Find the line "submission_url": "".
  3. Paste the Web app URL between the quotes: "submission_url": "https://script.google.com/macros/s/AKfyc.../exec"
  4. Save config.json.
Why this is a separate step: The URL is only known after you deploy the Apps Script in Google, so the initial publish leaves it blank for you to fill in.

9 Host the Form

The export folder is a self-contained static website. You can host it in two ways:

Option A — GitHub Pages

  1. Open your district GitHub account and open (or create) a Pages repository.
  2. Copy the contents of your export folder into a subfolder of the repo (for example, forms/2026-allergy-waiver/).
  3. Commit and push. GitHub will publish the files within a few minutes.
  4. The form URL will be something like https://eusd.github.io/forms/2026-allergy-waiver/.

Option B — Internal Web Server

  1. Connect to the district web server share (ask your tech coordinator for the path).
  2. Copy the export folder into the appropriate web-accessible directory.
  3. Confirm the URL with your tech coordinator and test it in a browser.

10 Submit a Test Entry

Before sending the form link to parents, do a quick test to make sure everything is connected.

  1. Open the live form URL in your browser.
  2. Fill out all the required fields with test data (e.g., student name "Test Student").
  3. Click Submit. A confirmation message should appear.
  4. Open your Google Sheet. A new row should appear at the bottom with the submission time and your test data.
  5. Check your Google Drive for a new folder named <form-id> submissions. Inside you should find a PDF and an HTML file for the test submission.
  6. Delete the test row from the Sheet before sharing the form link.

Understanding the Output

When you click Publish, Forms Creator writes the following to ~/projects/eusd/forms-creator/data/exports/<publish-path>/:

  • index.html — The form itself. Contains all field labels in both languages embedded as data attributes. Parents see whichever language they choose.
  • form.css / form.js — Shared styling and the language-switching and form-submission logic. Do not edit these files.
  • config.json — Machine-readable config the browser reads to know where to POST the submission. You must fill in submission_url before the form can submit.
  • apps_script.gs — The Google Apps Script source code. Paste this into your Sheet's Apps Script editor and deploy it as a web app.
What happens on submit: The browser sends the form data as JSON to your Apps Script URL. The script saves a row in the Sheet and creates a PDF + HTML file in a Google Drive folder. The browser shows the parent a confirmation.

Troubleshooting

ProblemCauseFix
python: command not found or 'python' is not recognized Python is not installed or not on the system PATH Install Python 3.10+ from python.org. During install, check "Add Python to PATH".
Browser shows "This site can't be reached" at localhost:5000 The server is not running Make sure python app.py is still running in a terminal window. If the terminal closed, start it again.
Activate.ps1 is blocked by execution policy Windows PowerShell default policy blocks scripts Run PowerShell as Administrator and run Set-ExecutionPolicy RemoteSigned. Then try again in a regular terminal.
Publish succeeds but the form doesn't submit (no Sheet row) submission_url in config.json is empty or incorrect Open config.json in your export folder, paste the Apps Script Web app URL into the submission_url field, save, and re-upload the file.
Apps Script authorization error on submit The Apps Script was not authorized, or the deployment was revoked In Google Sheets, go to Extensions → Apps Script, click Deploy → Manage deployments, and re-deploy. Re-authorize when prompted.
Sheet row appears but no PDF in Drive Drive storage may be full, or the script lacked Drive permission Check available Drive storage. Re-authorize the Apps Script (it requests Drive access). If the folder already exists with a different owner, rename it in Drive settings.
Spanish text shows garbled characters File was saved or uploaded with the wrong character encoding Always use UTF-8 when saving edited files. In Notepad, use File → Save As and set Encoding to UTF-8.
"Field type 'radio' requires non-empty options" when adding a field A radio or dropdown field was created without any options When adding a radio or dropdown field, use the Options section in the Add Field form to enter at least one option before saving.
Publish path validation error: "must be lowercase letters/digits/hyphens" The publish path contains spaces, uppercase letters, or special characters Use only lowercase letters, digits, and hyphens (e.g. spring-field-trip-2026). The path must start with a letter or digit.
Changes to a form were lost after restarting the server The data/forms.db file was deleted or corrupted Always stop the server cleanly (Ctrl+C in the terminal). Never delete data/forms.db while the server is running.
