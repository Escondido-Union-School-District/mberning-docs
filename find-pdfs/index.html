What This Tool Does

Find PDFs crawls pages on EUSD school websites, looking for 5il.co and aptg.co shortcode URLs embedded in the HTML. For each shortcode it finds, it follows the link and checks whether the destination is a PDF file.

Sites covered: the main district site (www.eusd.org) plus all 27 elementary, middle, and specialty school sites hosted under *.eusd.org.

What it produces:

  • A results.json file listing every shortcode found, the page it appeared on, the URL it resolves to, and whether it's a PDF
  • An interactive HTML viewer (results.html) for filtering, searching, and exporting the results
Note: The tool fetches raw HTML only — it does not execute JavaScript on the page. Shortcodes embedded in dynamically-loaded content may not be found.

Step-by-Step: find_pdfs.py

A basic scan from start to finish.

1 Open a terminal

Open Command Prompt, PowerShell, or bash and navigate to the project folder:

cd C:\Users\mberning\projects\eusd\find-pdfs

2 Run a scan

Scan a single site (up to 50 pages):

python find_pdfs.py --start https://www.eusd.org --domain eusd.org --max-pages 50

This crawls pages on www.eusd.org, finds all shortcodes, checks each one, and writes results.json.

3 View the results

Open results.html in any web browser. Click "Choose results.json" and select the output file.

Note: The results.html file and results.json should be in the same folder. The viewer will attempt to auto-load results.json from the same directory.

Providing Start URLs

You must provide at least one starting URL. There are three ways to do this:

Single URL

python find_pdfs.py --start https://www.eusd.org --domain eusd.org

Multiple URLs (comma-separated)

python find_pdfs.py --starts "https://www.eusd.org,https://bernardo.eusd.org" --domain eusd.org

URL list file

python find_pdfs.py --start-list starts.txt --domain eusd.org

The file should contain one URL per line. The project includes a starts.txt with all 27 EUSD school homepages.

Sitemap Options

Sitemaps help the crawler discover pages that may not be linked from the homepage. These URLs are added to the crawl queue alongside start URLs.

Specify a sitemap

python find_pdfs.py --start https://www.eusd.org --domain eusd.org --sitemap https://www.eusd.org/sitemap.xml

Multiple sitemaps

python find_pdfs.py --start https://www.eusd.org --domain eusd.org --sitemaps "https://www.eusd.org/sitemap.xml,https://bernardo.eusd.org/sitemap.xml"

Auto-detect sitemaps

python find_pdfs.py --start-list starts.txt --domain eusd.org --auto-sitemaps

This tries https://<host>/sitemap.xml for each start URL automatically.

Understanding the Output

JSON Output

The tool writes a JSON array to results.json (or the file specified by --out). Each entry looks like:

{
  "page": "https://bearvalley.eusd.org",
  "shortcode": "https://5il.co/35twf",
  "link_text": "Student Handbook",
  "final_url": "https://files-backend.assets.../Handbook.pdf",
  "status": 200,
  "is_pdf": true,
  "content_type": "application/pdf"
}
FieldDescription
pageThe EUSD page where the shortcode was found
shortcodeThe 5il.co or aptg.co URL
link_textInferred label from the HTML (may be empty if no label was found)
final_urlWhere the shortcode redirects to (empty on error)
statusHTTP status code (200, 404, etc.) or "error"
is_pdftrue if the destination is a PDF file
content_typeThe HTTP Content-Type header from the destination

Results Viewer

Open results.html in a browser and load your JSON file. The viewer provides:

  • Filter by type — All, PDF only, Non-PDF, or Errors
  • Search — filter by page URL, shortcode, or final URL
  • Hide noisy pages — checkbox to hide /documents, /news, /article pages
  • Export CSV — download results as a spreadsheet-ready CSV file
  • Visual tags — PDF, Error, and Non-PDF labels for quick scanning
  • Live counts — shows total results and PDF count for the current filter

Example Workflows

Scan a single school site

python find_pdfs.py --start https://bernardo.eusd.org --domain eusd.org --max-pages 100

Scan all 27 schools using the included start list

python find_pdfs.py --start-list starts.txt --domain eusd.org --max-pages 500

Scan with auto-detected sitemaps for deeper coverage

python find_pdfs.py --start-list starts.txt --domain eusd.org --auto-sitemaps --max-pages 1000

Debug a specific shortcode that isn't resolving

python find_pdfs.py --start https://www.eusd.org --domain eusd.org --debug --debug-shortcode "https://5il.co/35twf"

Save results to a custom file

python find_pdfs.py --start-list starts.txt --domain eusd.org --out march-2026-results.json

Troubleshooting

ErrorCauseFix
No start URLs provided No starting URL was supplied on the command line Provide at least one URL using --start, --starts, or --start-list
SSL/TLS certificate errors Certificate verification failed for a site Add --insecure to skip certificate checks (safe for internal EUSD crawling)
Crawl is very slow Too many pages, high delay, or poor network Reduce --max-pages, decrease --delay (minimum ~0.1), or check network connectivity
Many shortcodes show "status": "error" Shortcode service (5il.co / aptg.co) down, network issues, or timeout too low Run with --debug to see details; increase --timeout 30 if needed
Empty results (0 shortcodes found) Start URL has no shortcodes, --max-pages too low, or content is JavaScript-rendered Try --start-list starts.txt, increase --max-pages, or try a different start URL
Results viewer doesn't load data JSON file not loaded into the viewer Click "Choose results.json" and select your output file manually
