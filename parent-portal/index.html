What This Tool Does

The EUSD Parent Portal is a single-page website that gives parents quick access to school information, grades, calendars, lunch menus, enrollment, and more. It is a static site hosted on Netlify — there is no backend, no database, and no login required.

All content is driven by JSON data files in the data\ folder. To update the site, you edit those JSON files, commit the changes, and push to GitHub. Netlify automatically rebuilds and publishes the site.

Key concept: Everything is a JSON edit. You never need to touch the HTML or JavaScript to update school info, calendar dates, alert banners, or tray content. Just edit the right .json file and redeploy.

Prerequisites

One-Time Setup (already done on your machine)

  1. Git — to clone the repo, commit, and push changes
  2. A text editor — VS Code recommended (has built-in JSON validation)
  3. Node.js 18+ — only needed if you want to run the data scraper scripts
  4. GitHub access — push access to the eusd/parent-portal repository
  5. Project folder: C:\Users\mberning\projects\eusd\parent-portal

For routine content updates (alerts, principals, calendar dates), you only need Git and a text editor.

If Setting Up on a New Machine

# 1. Clone the repository
git clone https://github.com/eusd/parent-portal.git
cd parent-portal

# 2. Install scraper dependencies (optional — only for data scrapers)
npm install
npx playwright install chromium

Step-by-Step: Local Preview

Because this is a static site that loads JSON files via fetch(), you need a local web server. You cannot open index.html directly by double-clicking it.

1 Open a terminal in the project folder

cd C:\Users\mberning\projects\eusd\parent-portal

2 Start a local server

# Python (built into most systems)
python -m http.server 8000

# Node.js
npx serve .

# VS Code: install "Live Server" extension, right-click index.html → Open with Live Server

3 Open the site in your browser

Navigate to http://localhost:8000. You should see the full Parent Portal with all data loaded.

Warning: You cannot open index.html directly as a file (double-click). The page loads JSON data via fetch(), which requires a web server.

Step-by-Step: Common Maintenance Tasks

1 Post an Urgent Alert

Urgent alerts override the calendar banner at the top of the page. Use them for emergency closures, weather events, or time-sensitive notices.

Edit data\config.json and set the urgentAlert field:

{
  "lastUpdated": "2026-03-25",
  "urgentAlert": "All schools closed tomorrow due to severe weather",
  "currentSchoolYear": "2025-2026"
}

To deactivate, set urgentAlert back to null:

"urgentAlert": null

Commit, push, and Netlify will redeploy automatically. The banner reverts to showing the next calendar event.

Warning: Don't forget to clear the alert once the emergency is over. A stale alert erodes parent trust.

2 Update a Principal Name

Edit data\schools.json. Find the school by its id and change the principal and optionally principalEmail fields:

{
  "id": "bernardo",
  "name": "Bernardo Elementary",
  "principal": "New Principal Name",
  "principalEmail": "newprincipal@eusd.org",
  ...
}

Update lastUpdated in data\config.json, commit, and push.

3 Update Bell Schedules

Edit data\schools.json. Each school has a bellSchedule object:

"bellSchedule": {
  "regularStart": "8:00 AM",
  "regularEnd": "2:30 PM",
  "thursdayEnd": "12:15 PM",
  "minimumEnd": "12:10 PM"
}

Update the times as needed. If a school doesn't have a Thursday early dismissal, omit thursdayEnd.

4 Add or Edit Calendar Dates

Edit data\calendar.json. Each school year has a dates array. Add, remove, or edit entries:

{
  "date": "2026-09-07",
  "label": "Labor Day",
  "type": "holiday"
}

For multi-day events, add an endDate:

{
  "date": "2025-11-24",
  "endDate": "2025-11-28",
  "label": "Thanksgiving Recess",
  "type": "break"
}
TypeWhat It MeansCalendar Filter
holidayFederal/state holiday, no schoolHolidays
breakMulti-day recess (Thanksgiving, Winter, Spring)Breaks
minimum-dayEarly dismissal dayEarly dismissal
non-student-dayNo school for students (conferences, etc.)No school
first-last-dayFirst or last day of schoolFirst/Last day

5 Start a New School Year

Perform these updates each year, typically in July-August before school starts:

  1. Update currentSchoolYear in data\config.json (e.g., "2026-2027")
  2. Ensure the new year exists in data\calendar.json with all dates
  3. Update principal names in data\schools.json if any changed
  4. Update bell schedules if they changed
  5. Update SARC URLs (sarcUrl) in data\schools.json
  6. Update copyright year in index.html footer (January)
  7. Set lastUpdated in data\config.json

6 Edit a Task Tray's Content

Task tray content lives in data\tasks.json. Find the task by id and edit the trayHtml.en field. This is HTML content — use standard tags:

{
  "id": "grades",
  "title": { "en": "Check grades & attendance" },
  "trayHtml": { "en": "<p>Your updated content here...</p>" },
  ...
}
Warning: HTML inside JSON must have quotes escaped as \" and newlines removed. If the HTML is complex, write it in a separate .html file first, then paste it into the JSON as a single line with escaped quotes.

7 Update App Info

Edit data\apps.json. Each app has these key fields:

  • name.en — display name
  • description.en — short description
  • accessHtml.en — HTML instructions for how to access
  • loginUrl — main login/access link
  • appStoreUrl / playStoreUrl — mobile app links
  • helpUrl — "Having trouble?" link (use "#" as placeholder)

8 Refresh Staff Data

Staff data is scraped from each school's staff page on eusd.org. To refresh it:

node scripts\scrape-staff.js

This visits each school's /staff page, extracts names, roles, and emails, and writes the result to data\staff.json. Takes a few minutes to complete (23 schools).

9 Refresh the Forms List

Forms and PDFs are crawled from eusd.org:

node scripts\crawl-forms.js

This spider crawls up to 500 pages on eusd.org, collecting PDF links and form pages. Output goes to data\forms.json. Takes several minutes.

Note: After editing any data file, always update the lastUpdated field in data\config.json to today's date (YYYY-MM-DD format). This is shown in the site footer.

Step-by-Step: Deploying Changes

The site is hosted on Netlify and deploys automatically when you push to the main branch on GitHub.

1 Make your edits

Edit the relevant JSON file(s) in the data\ folder.

2 Preview locally

Start a local server and verify your changes look correct.

3 Commit and push

git add data\config.json data\schools.json
git commit -m "Update principal name for Bernardo Elementary"
git push

4 Verify the live site

Netlify typically deploys within 1-2 minutes. Visit the live site and confirm your changes appear.

Note: The CSS file is loaded with a version query string (e.g., styles.css?v=20260319d). If you make CSS changes and they don't appear on the live site, update the version string in index.html to bust the browser cache.

CLI Reference

The scraper scripts in the scripts\ folder are run with Node.js. These use Playwright because EUSD uses Thrillshare, which renders content client-side — simple HTTP fetches don't work.

# Install dependencies first (one time)
npm install
npx playwright install chromium

# Then run any script
node scripts\scrape-staff.js
node scripts\crawl-forms.js
node scripts\scrape-schools.js
node scripts\scrape-bells.js
Warning: Scrapers depend on the structure of eusd.org pages. If EUSD redesigns their site, scrapers may need updating. Check the console output for errors.

Understanding the Output

The Parent Portal is a single-page application. All content is rendered client-side by js\app.js using data from the JSON files in data\.

When a user visits the site, the browser fetches all JSON files and builds the page dynamically. The main sections are:

  • Alert banner — shows the next calendar event, or an urgent alert if one is set in config.json
  • School selector — 23 school tiles with brand colors from schools.json
  • "I need to..." tasks — expandable tiles driven by tasks.json
  • Apps & Logins — quick links from apps.json
  • Calendar — interactive calendar from calendar.json
  • Search — powered by Fuse.js, indexes all JSON data on page load
  • Google Translate — embedded widget for multilingual access

There is no build step, no compilation, and no server-side processing. Editing a JSON file and pushing to GitHub is all it takes to update the live site.

Troubleshooting

ErrorCauseFix
Site shows blank page or missing sections A JSON syntax error is preventing data from loading Open browser console (F12 → Console). Paste your JSON into jsonlint.com to find the exact error. Common mistakes: trailing commas, missing commas, unescaped quotes, single quotes instead of double quotes.
Alert banner not showing urgent alert urgentAlert is still null or changes haven't deployed Verify data\config.json has "urgentAlert": "Your message". Push to GitHub and wait 1-2 min. Hard-refresh with Ctrl+Shift+R.
Search isn't finding new content Fuse.js builds its index on page load; browser has cached old data Hard-refresh (Ctrl+Shift+R). Data files are cached for 1 hour (set in netlify.toml).
Scraper script fails or returns empty data EUSD may have changed their page structure, or the site is down Check eusd.org is accessible. Try running the script again. If page structure changed, selectors may need updating. Verify Playwright: npx playwright install chromium.
Page loads but looks wrong (no styles) CSS cache issue — browser loading old stylesheet Update the CSS cache-buster version in index.html. Find the <link> tag for styles.css and change the ?v= value to today's date.
Google Translate not appearing Widget loads from Google CDN; may be blocked by ad-blockers or network restrictions Disable ad-blockers and try in an incognito window.
404 / "File not found" on the live site This is a single-page site with no sub-routes All links should point to index.html or external URLs. There are no sub-pages.
School colors look wrong Each school has a color field in schools.json used for tile gradients Edit the hex color value in the school's entry. Use a valid 6-digit hex code (e.g., #72050b).
Need to validate JSON before pushing Want to catch errors early VS Code highlights JSON errors automatically. Or run: node -e "JSON.parse(require('fs').readFileSync('data\config.json','utf8'))". Or paste into jsonlint.com.
