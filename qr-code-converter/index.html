What This Tool Does

This tool updates QR codes inside a PDF document. It finds QR codes that point to video.link wrapper URLs, resolves each one to its actual YouTube video URL, generates new QR codes pointing directly to YouTube, and replaces them in the PDF.

The result is an updated PDF where all QR codes link directly to YouTube videos instead of going through the video.link intermediary.

Requirement: The computer must have internet access so the tool can look up the YouTube URLs behind each video.link address.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3 must be installed
  2. Project folder: C:\Users\mberning\projects\eusd\qr-code-converter
  3. Dependencies installed via pip: opencv-python, PyMuPDF, numpy, qrcode[pil], requests, Pillow

If Setting Up on a New Machine

# 1. Clone or copy the project folder to:
#    C:\Users\YOUR_USERNAME\projects\eusd\qr-code-converter

# 2. Open a terminal in the project folder and install dependencies:
pip install opencv-python PyMuPDF numpy "qrcode[pil]" requests Pillow

Step-by-Step

1 Open a terminal in the project folder

cd C:\Users\mberning\projects\eusd\qr-code-converter

2 Run the script with your PDF

python convert_qr_codes.py "path\to\your-file.pdf"

You can pass any PDF file. To specify a custom output filename, add it as a second argument:

python convert_qr_codes.py "input.pdf" "output.pdf"

If you omit the output filename, the tool creates one automatically by appending -UPDATED to the input name (e.g., input-UPDATED.pdf).

The tool will:

  1. Scan all pages for QR code images (200×200 or 200×250 pixels)
  2. Decode each QR code and filter for video.link URLs
  3. Fetch each video.link page to find the real YouTube URL
  4. Generate new QR codes pointing to YouTube
  5. Replace the old QR code images in the PDF
  6. Save the updated PDF

3 Collect the output

The updated PDF appears next to the input file with -UPDATED in the name, or at the custom output path you specified.

Understanding the Output

While running, the script prints progress to the console:

Opening my-file.pdf...
Scanning for QR code images...
Found 8 unique QR codes.

Resolving YouTube URLs...
xref     video.link URL                                YouTube URL
--------------------------------------------------------------------------------------------------------------
42       https://video.link/w/abc123                    https://www.youtube.com/watch?v=abc123
55       https://video.link/w/def456                    https://www.youtube.com/watch?v=def456
...

Replacing 8 QR code images in PDF...
Saving to my-file-UPDATED.pdf...
Done!

Each row in the table shows an image cross-reference ID (xref), the original video.link URL decoded from the QR code, and the resolved YouTube URL that will replace it.

If any URLs fail to resolve, the tool prints ERROR for that row and continues processing the remaining QR codes.

Troubleshooting

ErrorCauseFix
No QR codes found. Exiting. The PDF has no images matching the expected QR code dimensions (200×200 or 200×250 pixels), or no QR codes contain video.link URLs Verify you passed the correct PDF. If the QR codes have different dimensions, edit the size check in the extract_qr_codes() function.
FileNotFoundError The input PDF path does not exist Double-check the file path you passed as the first argument. If the path contains spaces, wrap it in quotes.
Could not find youtube_id in ... The video.link page structure has changed or the page no longer exists Open the video.link URL in a browser to check if it still works. If the page structure changed, the regex in resolve_youtube_url() may need updating.
ConnectionError or Timeout No internet connection or video.link is unreachable Check your internet connection. The tool uses a 15-second timeout per URL. Try again when the network is available.
ModuleNotFoundError: No module named 'cv2' Python dependencies are not installed Run: pip install opencv-python PyMuPDF numpy "qrcode[pil]" requests Pillow
N URLs failed to resolve Some video.link URLs could not be looked up This is a warning, not a fatal error. The tool still processes all successful URLs. Check the ERROR rows in the output for details on which URLs failed.
