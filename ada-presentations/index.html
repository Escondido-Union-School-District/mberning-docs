ADA Presentations

User Guide — Escondido Union School District

Last updated: 2026-04-29

What This Tool Does

ADA Presentations is a local Flask web app that converts PowerPoint (.pptx) presentations (typically exported from Google Slides via File → Download → Microsoft PowerPoint) into ADA / WCAG 2.1 AA compliant HTML slide decks. It reads the real structure of each slide via python-pptx, renders slide previews with headless LibreOffice, and uses the Claude Vision API only where structural data falls short — generating alt text for images without author-provided descriptions, summarizing charts, and decoding QR codes that the primary decoder can't read.

Two theme modes:

  • EUSD branding — uniform royal-blue/gold EUSD look, recommended for official presentations.
  • Preserve original — keeps the author's colors and fonts from the source deck, while still enforcing WCAG contrast ratios.
Requirements:
  • LibreOffice installed (used headlessly to render slide previews). Install via scoop install libreoffice or from libreoffice.org.
  • An API key in .env: either ANTHROPIC_API_KEY (for Claude) or GOOGLE_API_KEY (for Gemini, the default).

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.14 installed
  2. Project folder: ~/projects/eusd/ada-presentations/
  3. Dependencies installed via pip install -r requirements.txt
  4. LibreOffice installed (scoop install libreoffice or the installer from libreoffice.org)
  5. API key set in .env: GOOGLE_API_KEY=... (default) or ANTHROPIC_API_KEY=sk-ant-...

If Setting Up on a New Machine

# 1. Clone the repo
git clone https://github.com/Escondido-Union-School-District/ada-presentations.git
cd ada-presentations

# 2. Install Python dependencies
pip install -r requirements.txt

# 3. Install LibreOffice (one of the following)
scoop install libreoffice
# or download and install from https://www.libreoffice.org/download/

# 4. Create a .env file with an API key
echo "GOOGLE_API_KEY=..." > .env

Step-by-Step

1 Start the web app

python run.py

The app starts at http://localhost:5000.

2 Export your Google Slides deck as PPTX

In Google Slides: File → Download → Microsoft PowerPoint (.pptx).

3 Upload the .pptx

Open the app in your browser, drop the file onto the upload zone, and choose a theme mode (EUSD branding vs preserve original). A new project folder is created under ~/projects/eusd/ada-presentations/projects/.

4 Wait for extraction, rendering, and enrichment

The status page progresses through three stages: extracting slide structure via python-pptx, rendering slide previews via LibreOffice, and generating alt text / chart summaries / QR fallback via the AI Vision API. When done, the app redirects to the editor.

5 Review and edit

For each slide, review the original preview alongside the extracted blocks. Adjust headings, list items, table data, image alt text (with an Author/AI/Custom toggle), and speaker notes. Mark each slide reviewed with Ctrl+Enter and navigate with Arrow keys.

6 Export the accessible HTML deck

Click "Export Deck" to assemble a reveal.js slide deck. Preview it in-browser or download as a zip.

Understanding the Output

Each uploaded .pptx becomes a project folder under projects/ containing the original file, slide PNGs, extracted images (including any QR codes), a project.json holding all structural data and AI enrichment, and on export an output/ folder with the final HTML deck. The deck uses reveal.js with the selected theme (EUSD branding — royal blue #1B3A8C, accent blue #1a56db, highlight gold #f9a825, Arial font — or preserve-original with contrast enforcement) and meets WCAG 2.1 AA: semantic headings that never skip levels, alt text on every image, author-set descriptions honored first, accessible tables with scope attributes, keyboard navigation, speaker notes delivered via <aside class="notes">, and QR codes rendered as both image and clickable link.

Troubleshooting

ErrorCauseFix
LibreOfficeNotFoundError soffice binary not found at known locations or on PATH Install LibreOffice, or set LIBREOFFICE_PATH in .env to the full path of soffice.exe
API key not set Missing or empty .env file Create .env at project root with GOOGLE_API_KEY=... or ANTHROPIC_API_KEY=sk-ant-...
Only .pptx files are supported. Uploaded a PDF or other file type Re-export from Google Slides as .pptx and upload that
Address already in use Port 5000 is occupied by another process Stop the other process, or change the port in run.py
ModuleNotFoundError Dependencies not installed Run pip install -r requirements.txt
