EUSD Daily Digest

User Guide — Escondido Union School District

Last updated: 2026-04-16 — Initial guide created

What Is This?

EUSD Daily Digest is an automated tool that emails you a once-a-day summary of everything that happened across all your EUSD projects. It runs silently in the background at 6 AM every day and sends a single email to mberning@eusd.org — no action required on your part.

If you were away and your computer was off at 6 AM, the digest fires automatically the next time you log in, so you never miss a day.

What This Tool Does

Each morning, the digest scans every project under ~/projects/eusd/ and collects three types of activity from the previous calendar day:

CollectorWhat It CollectsWhere It Looks
Git commits All non-merge commits made during the day, with short hash, author, and subject Each project's .git folder via git log
Session notes Claude Code session notes saved during the day — what you were working on, decisions made, next steps ~/.claude/projects/C--Users-mberning-projects-eusd-<project>/memory/session-notes.md
Tool findings Any files modified during the day in reports/, output/, or logs/ directories Up to 2 levels deep inside each project's output folders

Projects with zero activity still appear in the email with an explicit No activity. note — silence is the signal that a project was untouched, not a bug.

Note: If you do not receive an email on any given morning, something broke. The digest is designed to always send, even when all projects are quiet.

Prerequisites

One-Time Setup (already done on your machine)

  1. Python 3.10+ is installed and on your PATH
  2. Project folder: ~/projects/eusd/eusd-daily-digest/
  3. Dependencies installed in the local venv/ (python-dotenv)
  4. Windows Scheduled Task named EusdDailyDigest created via register_task.ps1 — triggers at 6 AM daily and again at logon
  5. Network access to mail.eusd.org:25 (SMTP relay, no authentication needed)

If Setting Up on a New Machine

# 1. Copy or clone the project to:
#    C:\Users\mberning\projects\eusd\eusd-daily-digest\

# 2. Open a terminal in the project folder and create a virtual environment:
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt

# 3. Register the scheduled task (run once from the project folder):
powershell -ExecutionPolicy Bypass -File .\register_task.ps1
Warning: The scheduled task uses Interactive logon — it requires you to be logged in when it fires. It will not run while the machine is at the Windows login screen with no user session active.

Step-by-Step

Under normal operation the digest runs automatically and you do nothing. These steps are for manual runs, backfilling missed dates, or debugging.

1 Open a terminal in the project folder

cd ~/projects/eusd/eusd-daily-digest

2 Run the digest manually (sends email for yesterday)

python digest.py

You will see a confirmation line like Sent: EUSD daily digest — Tue Apr 15, 2026 if the email was sent successfully.

3 Preview output without sending an email

python digest.py --stdout

Prints both the plain-text and HTML versions to the terminal. Useful for checking what the email would contain before sending.

4 Backfill a specific date

python digest.py --date 2026-04-10

Sends the digest for April 10 instead of yesterday. Combine with --stdout to preview without sending.

5 Force re-send if today's digest was already sent

python digest.py --force

The digest records the last date it sent in a .last_sent file to avoid duplicates when the logon trigger fires multiple times. Use --force to override that guard.

CLI Reference

FlagDefaultDescription
--date YYYY-MM-DD yesterday Summarize a specific calendar day instead of yesterday
--no-email / --stdout off Print HTML and plain-text output to the terminal instead of sending via SMTP
--force off Bypass the .last_sent idempotency guard and send even if the target date was already sent today

  docs/
    user-guide.html      <-- This guide

Understanding the Output

The digest email has one section per EUSD project, sorted alphabetically. Each section shows up to three subsections:

  • Commits — short hash, author name, and commit subject for every non-merge commit on that day
  • Session notes — the full text of any Claude Code session note entries timestamped on that day
  • Tool findings — file path and a preview (up to 10 lines) of any output files modified on that day

If a project had no activity at all, the section reads No activity. If a collector encountered an error for a specific project, it shows ⚠ error: ... for that section only — other projects and other collectors still run normally.

The subject line header row summarizes totals across all projects, for example:

12 projects · 3 with commits · 1 with findings · 2 with session notes · 6 quiet

When the digest itself fails completely (not just one collector), you will receive a separate failure email with the full Python traceback so you can diagnose the problem.

Troubleshooting

Error / SymptomCauseFix
No email received this morning Machine was off at 6 AM and the logon trigger hasn't fired, or the task is disabled Log in — the logon trigger fires 2 minutes after login. Or run python digest.py manually. Check Task Scheduler if it keeps happening.
Already sent digest for YYYY-MM-DD; skipping. The logon trigger fired but the digest was already sent by the 6 AM trigger (normal behavior) Not an error — this is the idempotency guard working correctly. Use --force if you actually need to re-send.
git not on PATH in one project section Git is not installed or not in the PATH visible to the scheduled task's environment Install Git for Windows and ensure it is on the system PATH. Re-register the task after fixing.
git log timed out after 30s A project's git repo is very large or the drive is slow Check if the repo has excessive history or is on a network drive. The timeout is per-project so other projects still run.
Session notes section empty for a project you worked on The session notes file path doesn't match the expected slug format, or notes weren't saved via the "eod" command Check that ~/.claude/projects/C--Users-mberning-projects-eusd-<project-name>/memory/session-notes.md exists and has timestamped entries. Use the eod command in Claude Code to save notes.
Failure email with ConnectionRefusedError or SMTP error Cannot reach mail.eusd.org:25 — you may be off-network or VPN is disconnected Connect to the EUSD network or VPN, then run python digest.py --force to retry.
ModuleNotFoundError: No module named 'dotenv' The virtual environment is not activated, or dependencies were not installed Run venv\Scripts\activate then pip install -r requirements.txt. The scheduled task uses run_hidden.vbs which points to the venv Python automatically.
Digest appears to hang or never finish A subprocess (e.g., git) is blocking; the watchdog will kill the process after 5 minutes Wait up to 5 minutes for the watchdog to terminate the process. Then check log.txt for the last output before the hang. You can also run cleanup_stale.ps1 to kill any orphaned python.exe processes immediately.
Tool findings show <binary file, not previewed> A file in reports/, output/, or logs/ is a binary (PDF, image, etc.) Expected behavior — binary files are detected by null bytes and skipped to avoid garbled output. The file path and size are still shown.
