What This Tool Does

EUSD Newsletters is a district-hosted replacement for Smore for creating school newsletters. Staff build a newsletter from titled sections (text, photos, photo galleries, PDF flyers, and links), publish it to a stable web link, and share that link with families through their existing email tool. Every published page is built to be fully accessible (WCAG 2.1 AA), and each newsletter comes with detailed viewership analytics — how many people opened it, how far they read, and which links and flyers they clicked.

Prerequisites

One-Time Setup

  1. An @eusd.org Google account.
  2. Membership in your school's newsletter Google Group (this is what grants you access to your school's newsletters). Ask your site administrator if you are unsure.
  3. A current web browser (Chrome, Edge, Firefox, or Safari). No install needed.

If You Can't Sign In or Don't See Your School

You are signed in but no schools appear: you have not been added to a school's Google Group yet. Contact your site administrator to be added.

Step-by-Step

The intended workflow once the tool is live:

1 Sign in

Go to https://newsletters.eusd.org and sign in with your EUSD Google account.

2 Pick your school

You'll see the schools you have access to. Choose one.

3 Start a newsletter

Start blank, or duplicate your last issue to reuse the same sections.

4 Build it from sections

Add titled sections and drag them into the order you want. In each section add content blocks — paragraphs, links, images, photo galleries, and PDF flyers — and reorder them by dragging. Your work autosaves as you go.

Live now: titled sections; paragraph, link, image, image-gallery, and flyer (PDF) blocks; drag-to-reorder for sections and blocks; autosave; and a soft “someone else is editing this section” lock so two people don't overwrite each other (now enforced on the server, not just hidden in the screen). Coming in Phase 3b: the content accessibility engine that checks and auto-fixes content as you write.

5 Add photos and flyers

Add an image block (single photo, or wrapped left/right), an image gallery (2–4 photos across), or a flyer (PDF) block, then use its Upload button to attach a file. Photos are processed automatically — location/camera data is stripped and fast web-sized versions are generated for you.

  • Alt text is required on every image. Type a short description of what the photo shows. Click Generate alt text to get an AI-drafted suggestion — you must read it and accept or edit it; it is never saved automatically.
  • Flyers get an accessibility check. Each PDF flyer shows a status of unchecked, pass, or fail. Run the check on the flyer block; a flyer that fails (or is left unchecked) will block publishing in Phase 4.
  • Extract text from a flyer. Use Extract text from this flyer to pull the PDF's words — and any QR-code links — into the section as editable paragraph and link blocks, so the flyer's information is also readable as real text. Review and tidy the extracted blocks (and give each pulled-in link a clear description) before publishing.

6 Preview, then publish

Mark the newsletter published when it's ready and share its link with families; mark it back to draft to take it offline.

Phase 4 note: the full public page and the two-tier accessibility publish gate arrive in Phase 4. Today, publishing is a draft/published status switch — the polished public reader page is not built yet.

7 Watch the analytics

Open a newsletter's analytics for views, reads, and clicks, or the school dashboard for trends across all your issues.

Troubleshooting

ProblemCauseFix
Signed in but no schools listed Not in any school's Google Group Ask your site administrator to add you to the school group
A section is read-only and shows “Another editor is editing this section” A colleague has that section open; it is soft-locked to prevent overwrites Edit a different section, or wait — the lock releases automatically after a period of inactivity, then reload the page
Can't publish — blocked An accessibility check failed (e.g., image with no alt text, inaccessible PDF) Fix the listed items; the publish screen shows exactly where each problem is (the publish accessibility gate arrives in Phase 4)
Can't save an image block The alt-text field is empty Type a short description of the photo (or use Generate alt text and accept/edit the suggestion) — alt text is required on every image
Generate alt text says to write it manually The AI service is unavailable or the image file is missing Type the alt text yourself describing what the photo shows; try the button again later if you prefer a suggestion
A flyer still shows “unchecked” The accessibility check hasn't been run on that flyer yet Run the check on the flyer block; if it fails, replace the PDF with a tagged, accessible version (an unchecked or failing flyer blocks publishing)
The Upload button does nothing JavaScript is turned off in your browser Turn JavaScript on (uploading needs it), or ask a colleague with it enabled to attach the file
