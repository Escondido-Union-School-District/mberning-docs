What This Tool Does

Smore ADA Checker audits Smore newsletter posts for accessibility issues. It uses a headless browser to scrape the page, runs 15+ automated checks (including AI-powered image and link analysis), and produces a detailed report with screenshots and suggested fixes.

StageWhat HappensYour Role
1. Scrape & Check Playwright loads the Smore page, scrapes content, runs all accessibility checks, captures screenshots Provide the Smore URL and wait
2. Review A local web page opens showing all found issues with toggles and editable suggestions Include/exclude issues, edit alt text and suggestions, then generate report or send email
Note: If no accessibility issues are found, the PDF generates automatically without the review step.
Finding Smore URLs: The Thrillshare Checker project (C:\Users\mberning\projects\eusd\thrillshare-checker) has a --smore-scan mode that automatically finds Smore links in principal emails on Thrillshare. It runs daily at 7:00 AM and emails you any new links. Copy the URLs from that email and paste them here to run the ADA check.

What Gets Checked

CategoryChecks
Images Missing alt text, filename as alt text, alt text too long (>200 chars), ineffective alt text (AI vision), duplicate alt text across images
Flyers Important flyer info (dates, times, locations) not in surrounding text, QR codes without text link alternatives
Links Generic text ("click here"), vague text ("more info here"), raw URLs as link text, instructional phrasing ("click here to...", "fill out the survey"), AI evaluation of all other links for descriptiveness (WCAG 2.4.4), links to untagged file downloads (PDF, DOCX, PPTX), Google Workspace links
Headings Skipped heading levels (e.g., H2 jumps to H4)
Emoji Emoji in headings, consecutive emojis, emojis used as bullet points, emoji at start of text
Video Direct video file links (no captions), non-YouTube video embeds

  1. Python 3.10+ installed and on your PATH
  2. Project folder: C:\Users\mberning\projects\eusd\smore-ada-checker

Step-by-Step

1 Open a terminal in the project folder

cd C:\Users\mberning\projects\eusd\smore-ada-checker
venv\Scripts\activate

2 Run the checker on a Smore URL

python check.py https://app.smore.com/n/jtycf

The tool will launch a headless browser, scrape the page, and run all accessibility checks. This typically takes 30–90 seconds depending on the number of images.

You'll see progress output like:

[Scraping] Loading https://app.smore.com/n/jtycf ...
[Checks]   Running non-vision checks...
[Vision]   Classifying 8 images (2 batches)...
[Vision]   Evaluating alt text for 3 images...
[Vision]   Comparing flyer content for 2 flyers...
[Done]     Found 12 issues

3 Review issues in the browser

If issues are found, a review page opens automatically at http://localhost:5050. Here you can:

  • Toggle issues — uncheck any issue to exclude it from the report
  • Edit suggestions — all issue types (alt text, link text, flyer details, emoji, heading, and video messages) have an editable field. Image alt text cards show the current alt text in an amber box and the AI suggestion in a green box. A Copy button next to each suggestion copies it to the clipboard in one click.
  • Add an auditor note — every issue card has an optional note field at the bottom. Notes appear as a cream callout in the staff email and PDF, useful for giving the newsletter author extra context or pointing them to a specific section.
  • View screenshots — click any screenshot to enlarge it in a lightbox
  • AI prompt tips — a single panel at the top of the page provides copyable prompts for writing alt text (upload the image to Claude/ChatGPT/Gemini with the prompt) and extracting flyer content (run the prompt on the flyer image to get text you can paste into the newsletter body)

4 Generate the report

At the bottom of the review page, choose one of:

  • Generate Final Report — saves a PDF to the reports\ folder
  • Send Preview to Me — emails an HTML report to your auditor email for review, then offers a "Send to Staff" button to forward to the newsletter author. An optional "Coming Soon: automated fixes" banner can be included by checking the checkbox before previewing (unchecked by default)
Warning: Do not close the terminal while the review page is open. The review server runs in that terminal session and will stop if the terminal closes.

5 Find your report

PDF reports are saved to:

reports\{newsletter-title-slug}-{smore-code}-{date}-{provider}.pdf

For example: reports\coyote-connection-r1y7x-2026-05-14-claude.pdf

Every time you click Send to Staff, a row is automatically appended to reports\email-log.csv with the timestamp, recipients, issue count, your note, and the PDF filename. Open it in Excel for a paper trail — useful if you need to report to the director which principals have received reports and haven't responded.

Understanding the Output

PDF Report

The PDF report contains:

  • Summary dashboard — total issue count and breakdown by category
  • Issues grouped by section — each issue shows:
    • Color-coded category badge (image, link, heading, etc.)
    • Plain English description of the problem
    • Current value (the alt text, link text, or missing details)
    • Suggested fix (your edited suggestion from the review step)
    • AI prompt tip for alt text and flyer issues — a ready-made prompt staff can paste into ChatGPT, Claude, or Gemini along with the image to write their own alt text or extract flyer content
    • Screenshot of the issue on the page

HTML Email Report

The email version contains the same information as the PDF, formatted for Gmail with inline images. It includes:

  • Page title and URL at the top, with a stats card showing issue counts and estimated fix time
  • Optional notes-to-staff message (entered during review)
  • All included issues with embedded screenshots and any auditor notes you typed during review
  • A single AI prompt tips panel (alt text + flyer prompts side by side) near the top of the report
  • A prominent green “Mark ALL Fixes as Complete” card at the top and bottom — when the recipient clicks it, the completion is logged in the Google Sheet (requires EMAIL_LOG_SHEET_WEBHOOK to be configured) and a green pill appears on the dashboard’s Sent row within 5 minutes

Vision Cache

API results are cached in smore_checker\vision_cache.json. Re-running the same URL reuses cached results, saving time and API costs. Use --clear-cache to force fresh analysis. Results are stored per-provider, so switching between Claude and Gemini always produces independent results.

Troubleshooting

ErrorCauseFix
Error: URL must be a Smore page (smore.com) The URL you provided doesn't contain smore.com Use the full Smore URL, e.g., https://app.smore.com/n/jtycf
Cannot save PDF — file is open A PDF viewer has the report file locked Close the PDF in your viewer, then click the "back" link on the error page to retry
Playwright timeout or crash Browser failed to load the Smore page Run playwright install chromium to reinstall the browser, then try again
API key errors or 401 Unauthorized Missing or invalid API key in .env Check that ANTHROPIC_API_KEY (or GOOGLE_API_KEY) is set correctly in your .env file
Rate limit errors (429) Too many API calls in a short time Wait a minute and retry. Gemini free tier has stricter limits; the tool adds automatic delays
Review page won't load Port 5050 is already in use Close any other instance of the checker, or check if another program is using port 5050
Email fails to send SMTP configuration is incorrect Verify SMTP_HOST, SMTP_PORT, SENDER_EMAIL, and SMTP_PASSWORD in .env. For EUSD relay, leave password empty
ModuleNotFoundError Virtual environment not activated or dependencies missing Run venv\Scripts\activate then pip install -r requirements.txt

Quick Reference Card

Daily Workflow

  1. Open terminal, cd to project folder, activate venv
  2. Run: python check.py https://app.smore.com/n/XXXXX
  3. Wait for scraping and checks to complete (30–90 seconds)
  4. Review issues in browser — toggle and edit as needed
  5. Click Send Preview to Me to review the email, or Generate Final Report for PDF
  6. Find PDF in reports\ folder

Common Flags

  • --clear-cache — re-analyze images with fresh API calls
  • --no-review — skip review, generate PDF immediately
  • --verbose — see detailed API logging
